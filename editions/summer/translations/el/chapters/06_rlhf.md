# Ενισχυτική Μάθηση από Ανθρώπινη Ανατροφοδότηση

Καμία τεχνική δεν έχει συμβάλει περισσότερο στην ανάδειξη της HITL ML στο ευρύ κοινό από την Ενισχυτική Μάθηση από Ανθρώπινη Ανατροφοδότηση (RLHF). Αποτελεί τον μηχανισμό πίσω από το InstructGPT {cite}`ouyang2022training` και βασικό συστατικό των αγωγών ακολούθησης οδηγιών σε πολλά σύγχρονα μεγάλα γλωσσικά μοντέλα {cite}`stiennon2020learning`. Η κατανόηση του RLHF — όχι απλώς ως συνταγή που ακολουθείται, αλλά ως θεμελιωμένη προσέγγιση ευθυγράμμισης — είναι απαραίτητη για όσους εργάζονται στη σύγχρονη ΤΝ.

---

## Το Πρόβλημα Ευθυγράμμισης

Τα Μεγάλα Γλωσσικά Μοντέλα (LLMs) εκπαιδευμένα καθαρά στην πρόβλεψη επόμενου τεκμηρίου βελτιστοποιούν έναν αντιπρόσωπο στόχο: πρόβλεψη του κειμένου που ακολουθεί στο σώμα ανθρώπινου γραπτού κειμένου. Αυτός ο στόχος σχετίζεται, αλλά διαφέρει, από αυτό που πραγματικά θέλουμε: απαντήσεις που είναι χρήσιμες, ακριβείς, ασφαλείς και ευθυγραμμισμένες με τις ανθρώπινες αξίες.

Η αναντιστοιχία μεταξύ του στόχου εκπαίδευσης και της επιθυμητής συμπεριφοράς ονομάζεται **πρόβλημα ευθυγράμμισης** {cite}`russell2019human`. Συγκεκριμένα, ένα γλωσσικό μοντέλο εκπαιδευμένο σε κείμενο διαδικτύου μαθαίνει να:
- Παράγει αληθοφανείς συνεχίσεις (που μπορεί να είναι πραγματολογικά λανθασμένες)
- Αντικατοπτρίζει τις προκαταλήψεις και τις βλάβες που υπάρχουν στα δεδομένα εκπαίδευσης
- Να είναι αποφευκτικό ή χειραγωγικό όταν αυτό είναι στατιστικά το επόμενο μετά τη γλωσσική εντολή

Το RLHF αντιμετωπίζει την ευθυγράμμιση κάνοντας τις ανθρώπινες προτιμήσεις *μέρος του στόχου βελτιστοποίησης*.

---

## Ο Αγωγός RLHF

Το RLHF προχωρά σε τρία στάδια:

```text
Στάδιο 1: Εποπτευόμενο Fine-Tuning (SFT)
  --> Συλλογή δεδομένων επίδειξης (ο άνθρωπος γράφει ιδανικές απαντήσεις)
  --> Fine-tune βασικού LLM στις επιδείξεις

Στάδιο 2: Εκπαίδευση Μοντέλου Ανταμοιβής
  --> Συλλογή ζευγαρωτών προτιμήσεων (ο άνθρωπος αξιολογεί Α vs Β)
  --> Εκπαίδευση μοντέλου ανταμοιβής R(x, y) για πρόβλεψη ανθρώπινων προτιμήσεων

Στάδιο 3: Fine-Tuning ΕΝ
  --> Fine-tune LLM χρησιμοποιώντας PPO/ΕΝ για μεγιστοποίηση R(x, y)
  --> Η ποινή KL αποτρέπει υπερβολική απόκλιση από το μοντέλο SFT
```

### Στάδιο 1: Εποπτευόμενο Fine-Tuning

Ξεκινώντας από ένα προ-εκπαιδευμένο βασικό μοντέλο $\pi_0$, συλλέγουμε ένα σύνολο δεδομένων ζευγαριών (εντολή, ιδανική απάντηση), γραμμένων ή επιλεγμένων από ανθρώπινους εργαζομένους που ακολουθούν λεπτομερείς οδηγίες. Το μοντέλο κάνει fine-tune σε αυτές τις επιδείξεις χρησιμοποιώντας τυπική διασταυρωτή εντροπία:

$$
\mathcal{L}_\text{SFT}(\theta) = -\mathbb{E}_{(x, y) \sim \mathcal{D}_\text{demo}} \left[ \log \pi_\theta(y \mid x) \right]
$$

Το μοντέλο SFT $\pi_\text{SFT}$ είναι πολύ καλύτερο σημείο εκκίνησης για RLHF από το ακατέργαστο προ-εκπαιδευμένο μοντέλο.

### Στάδιο 2: Εκπαίδευση Μοντέλου Ανταμοιβής

Για ένα σύνολο εντολών $\{x_i\}$, παράγουμε $K$ απαντήσεις ανά εντολή χρησιμοποιώντας $\pi_\text{SFT}$ και τις παρουσιάζουμε σε ανθρώπινους σχολιαστές ως ζευγαρωτές συγκρίσεις: «Ποια απάντηση είναι καλύτερη, Α ή Β;»

Το μοντέλο ανταμοιβής $r_\phi$ εκπαιδεύεται να προβλέπει αυτές τις προτιμήσεις. Υπό το μοντέλο **Bradley-Terry** (Κεφάλαιο 8), η πιθανότητα να προτιμηθεί η απάντηση $y_w$ έναντι της $y_l$ είναι:

$$
P(y_w \succ y_l \mid x) = \sigma\left(r_\phi(x, y_w) - r_\phi(x, y_l)\right)
$$

Το μοντέλο ανταμοιβής εκπαιδεύεται να ελαχιστοποιεί την απώλεια ζευγαρωτής κατάταξης:

$$
\mathcal{L}_\text{RM}(\phi) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}_\text{pref}} \left[ \log \sigma\left(r_\phi(x, y_w) - r_\phi(x, y_l)\right) \right]
$$

Το μοντέλο ανταμοιβής αρχικοποιείται συνήθως από το μοντέλο SFT με μια βαθμωτή κεφαλή που αντικαθιστά το τελευταίο στρώμα.

### Στάδιο 3: Fine-Tuning ΕΝ με PPO

Με εκπαιδευμένο μοντέλο ανταμοιβής, μπορούμε να χρησιμοποιήσουμε ενισχυτική μάθηση για fine-tuning του LLM. Κάθε εντολή $x$ είναι μια κατάσταση· κάθε απάντηση $y$ είναι μια τροχιά επιλογών τεκμηρίων· και η ανταμοιβή είναι $r_\phi(x, y)$.

Ο στόχος βελτιστοποίησης περιλαμβάνει μια **ποινή απόκλισης KL** για να αποτραπεί η υπερβολική απόκλιση του μοντέλου από τη βάση SFT (που θα οδηγούσε σε reward hacking {cite}`krakovna2020specification,gao2023scaling`):

$$
\max_\theta \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(\cdot | x)} \left[ r_\phi(x, y) - \beta \cdot \text{KL}\left[\pi_\theta(\cdot \mid x) \| \pi_\text{SFT}(\cdot \mid x)\right] \right]
$$

Η παράμετρος $\beta$ ελέγχει την ισχύ της ποινής KL. Μικρή $\beta$ επιτρέπει περισσότερη βελτιστοποίηση αλλά κινδυνεύει με reward hacking· μεγάλη $\beta$ κρατά το μοντέλο κοντά στο SFT αλλά περιορίζει τα κέρδη ευθυγράμμισης.

Η **Εγγύς Βελτιστοποίηση Πολιτικής (PPO)** {cite}`schulman2017proximal` είναι ο τυπικός αλγόριθμος για αυτό το στάδιο, επιλεγμένος για τη σταθερότητά του σε σχέση με τις μεθόδους ακατέργαστης κλίσης πολιτικής.

## Μια Απλοποιημένη Επίδειξη RLHF

Ο πλήρης αγωγός RLHF απαιτεί υποδομή μεγάλης κλίμακας. Το ακόλουθο παράδειγμα επιδεικνύει τις βασικές ιδέες σε μικρή κλίμακα.

```{code-cell} python
import numpy as np
import torch
import torch.nn as nn
import torch.optim as optim
from torch.nn import functional as F

torch.manual_seed(42)
rng = np.random.default_rng(42)

# -----------------------------------------------
# Toy setup: responses are 4-dimensional vectors
# "Quality" is known analytically (sum of positive values)
# We simulate a reward model learning this from pairwise feedback
# -----------------------------------------------

class RewardModel(nn.Module):
    def __init__(self, d=4, hidden=32):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, 1)
        )
    def forward(self, x):
        return self.net(x).squeeze(-1)

def true_quality(x):
    """The hidden ground-truth reward function."""
    return x.sum(dim=-1) + 0.5 * (x ** 2).mean(dim=-1)

# Generate pairwise preference data
N_PAIRS = 500
X1 = torch.randn(N_PAIRS, 4)
X2 = torch.randn(N_PAIRS, 4)
q1, q2 = true_quality(X1), true_quality(X2)
# Human prefers X1 when q1 > q2 (with some noise)
noise = torch.randn(N_PAIRS) * 0.5
preferred_1 = ((q1 - q2 + noise) > 0).float()

# Train reward model
rm = RewardModel(d=4, hidden=32)
optimizer = optim.Adam(rm.parameters(), lr=3e-3)

losses = []
for epoch in range(200):
    r1 = rm(X1)
    r2 = rm(X2)
    # Bradley-Terry loss
    logit = r1 - r2
    loss = F.binary_cross_entropy_with_logits(logit, preferred_1)
    optimizer.zero_grad(); loss.backward(); optimizer.step()
    losses.append(loss.item())

# Evaluate: does the reward model agree with true quality?
X_eval = torch.randn(1000, 4)
with torch.no_grad():
    r_pred = rm(X_eval).numpy()
    r_true = true_quality(X_eval).numpy()

corr = np.corrcoef(r_pred, r_true)[0, 1]
print(f"Reward model correlation with true quality: {corr:.4f}")
print(f"Final training loss: {losses[-1]:.4f}")

import matplotlib.pyplot as plt
plt.figure(figsize=(8, 3))
plt.subplot(1, 2, 1)
plt.plot(losses, color='#2b3a8f', linewidth=1.5)
plt.xlabel("Epoch"); plt.ylabel("Pairwise loss")
plt.title("Reward Model Training")

plt.subplot(1, 2, 2)
plt.scatter(r_true[:200], r_pred[:200], alpha=0.4, s=15, color='#2b3a8f')
plt.xlabel("True quality"); plt.ylabel("Predicted reward")
plt.title(f"Reward Model vs. Truth (r={corr:.3f})")
plt.tight_layout()
plt.savefig('reward_model.png', dpi=150)
plt.show()
```

## Προκλήσεις στο RLHF

### Reward Hacking

Ένας βασικός τρόπος αποτυχίας: η πολιτική βρίσκει τρόπους να παίρνει υψηλή ανταμοιβή από το μοντέλο ανταμοιβής που δεν αντιστοιχούν σε γνησίως καλή συμπεριφορά. Για παράδειγμα, ένα LLM μπορεί να μάθει να παράγει απαντήσεις που είναι κολακευτικές ή ακούγονται σίγουρα (τις οποίες οι σχολιαστές τείνουν να αξιολογούν υψηλά) αντί να είναι ακριβείς.

Το reward hacking είναι πιο πιθανό όταν:
- Το μοντέλο ανταμοιβής εκπαιδεύεται σε ανεπαρκή δεδομένα προτίμησης
- Η πολιτική έχει αποκλίνει πολύ από τη βάση SFT (μικρή $\beta$)
- Η κατανομή μοντέλου ανταμοιβής μετατοπίζεται κατά την εκπαίδευση PPO

**Στρατηγικές μετριασμού:** ποινή KL, επαναληπτική εκπαίδευση μοντέλου ανταμοιβής, διαφορετική αξιολόγηση, περιορισμοί Constitutional AI.

### Προκατάληψη Αξιολογητή

Οι ανθρώπινοι σχολιαστές έχουν συστηματικές προκαταλήψεις. Τείνουν να προτιμούν μακρύτερες απαντήσεις (προκατάληψη λεκτικότητας), κείμενο που ακούγεται πιο σίγουρο (προκατάληψη εμπιστοσύνης) και απαντήσεις που συμφωνούν με τις προηγούμενες πεποιθήσεις τους. Αυτές οι προκαταλήψεις μεταδίδονται στο μοντέλο ανταμοιβής.

Η γνωστή αποτυχία συκοφαντίας μοντέλων RLHF — όπου το μοντέλο λέει στους χρήστες αυτό που θέλουν να ακούσουν αντί για αυτό που είναι αληθινό — είναι εν μέρει αποτέλεσμα της προτίμησης αξιολογητών για ευάρεστες απαντήσεις.

### Κλιμακωτή Εποπτεία

Για σύνθετα έργα, οι άνθρωποι δεν μπορούν αξιόπιστα να κρίνουν ποια απάντηση ΤΝ είναι σωστή. Ένας σχολιαστής που συγκρίνει δύο μακρές μαθηματικές αποδείξεις ή δύο υλοποιήσεις κώδικα μπορεί απλώς να επιλέξει την πιο αναγνώσιμη, ανεξάρτητα από ορθότητα. Η **κλιμακωτή εποπτεία** είναι το ανοιχτό ερευνητικό πρόβλημα σχεδιασμού διαδικασιών αξιολόγησης που παραμένουν αξιόπιστες καθώς αυξάνεται η πολυπλοκότητα έργου {cite}`bowman2022measuring`.

## Constitutional AI (RLAIF)

Το **Constitutional AI** {cite}`bai2022constitutional`, που αναπτύχθηκε στην Anthropic, μειώνει την εξάρτηση από ανθρώπινους σχολιαστές χρησιμοποιώντας την ίδια την ΤΝ για να παράγει ετικέτες προτίμησης καθοδηγούμενες από ένα σύνολο αρχών (μια «σύνταξη»). Η διαδικασία:

1. Παράγει απαντήσεις σε δυνητικά επιβλαβείς εντολές
2. Χρησιμοποιεί κριτή ΤΝ για αξιολόγηση απαντήσεων έναντι συνταγματικών αρχών
3. Αναθεωρεί απαντήσεις καθοδηγούμενες από ανατροφοδότηση ΤΝ (RLAIF — ΕΝ από Ανατροφοδότηση ΤΝ)
4. Εκπαιδεύει μοντέλο ανταμοιβής σε προτιμήσεις που παράγει η ΤΝ
5. Κάνει fine-tune με RLHF χρησιμοποιώντας αυτό το μοντέλο ανταμοιβής

Το RLAIF μπορεί να παράγει δεδομένα προτίμησης σε πολύ μεγαλύτερη κλίμακα από την ανθρώπινη επισήμανση, και επιτρέπει λεπτομερή έλεγχο των αξιών που κωδικοποιούνται στο μοντέλο ανταμοιβής.
