# Αξιολόγηση και Μετρικές

Για να ξέρετε αν το σύστημά σας HITL λειτουργεί, χρειάζεστε περισσότερα από τη μέτρηση της ακρίβειας μοντέλου. Χρειάζεστε να γνωρίζετε αν αποκομίζετε αξία από τον προϋπολογισμό σχολιασμού σας, αν το μοντέλο είναι πραγματικά καλύτερα ευθυγραμμισμένο με την ανθρώπινη πρόθεση, και αν η πρόσθετη ανθρώπινη ανατροφοδότηση θα συνεχίσει να βελτιώνει τα πράγματα. Αυτό το κεφάλαιο καλύπτει το πλήρες τοπίο αξιολόγησης σε ρυθμίσεις HITL.

---

## Μετρικές Κεντρικές στο Μοντέλο

Οι τυπικές μετρικές ΜΜ εφαρμόζονται απευθείας σε συστήματα HITL, με ορισμένες σημαντικές αποχρώσεις.

### Μετρικές Ταξινόμησης

Η **ακρίβεια** είναι κατάλληλη όταν οι κλάσεις είναι ισορροπημένες και όλα τα σφάλματα κοστίζουν εξίσου. Σε ρυθμίσεις HITL, ωστόσο, το επισημασμένο σύνολο ελέγχου μπορεί να είναι μεροληπτικό από τη στρατηγική ερωτήματος (η ενεργός μάθηση υποβάλλει ερώτηση σε μη τυχαία παραδείγματα), καθιστώντας τις απλές εκτιμήσεις ακρίβειας αναξιόπιστες.

Η **βαθμολογία F1** είναι ο αρμονικός μέσος ακρίβειας και ανάκλησης, κατάλληλος για ανισόρροπες κλάσεις. Σε πλαίσια HITL, τόσο η ακρίβεια όσο και η ανάκληση μπορεί να έχουν διαφορετική σημασία ανάλογα με την ασυμμετρία κόστους μεταξύ ψευδών θετικών και ψευδών αρνητικών.

Το **AUROC** μετρά την ικανότητα του μοντέλου να διακρίνει μεταξύ κλάσεων ανεξάρτητα από κατώφλι — σημαντικό για εργασίες ευαίσθητες στη βαθμονόμηση, όπως η ιατρική διαλογή.

Η **βαθμονόμηση** μετρά πόσο καλά οι προβλεπόμενες πιθανότητες αντιστοιχούν σε εμπειρικές συχνότητες. Σε συστήματα HITL, τα μοντέλα εκπαιδευμένα σε μεροληπτικά επισημασμένα σύνολα (από ενεργό μάθηση) μπορεί να είναι κακώς βαθμονομημένα ακόμα και όταν είναι ακριβή.

### Μετρικές Γενεστικών Μοντέλων

Για γλωσσικά μοντέλα και γενεστικά συστήματα, η αξιολόγηση είναι θεμελιωδώς δυσκολότερη. Καμία μεμονωμένη αυτόματη μετρική δεν αποτυπώνει ποιότητα:

- **BLEU / ROUGE / METEOR:** Μετρικές βάσει αναφοράς για μετάφραση και περίληψη. Συσχετίζονται αδύναμα με ανθρώπινες κρίσεις ποιότητας για εκτεταμένη δημιουργία.
- **Perplexity:** Μετρά πόσο καλά το μοντέλο προβλέπει κείμενο που κρατήθηκε εκτός εκπαίδευσης. Αναγκαία αλλά όχι επαρκής συνθήκη για ποιότητα.
- **BERTScore:** Ομοιότητα βάσει ενσωμάτωσης με αναφορές. Καλύτερα συσχετισμένη με ανθρώπινες κρίσεις από μετρικές n-gram.
- **Ανθρώπινη αξιολόγηση:** Το χρυσό πρότυπο. Βλ. Ενότητα 14.3.

## Μετρικές Αποδοτικότητας Σχολιασμού

Η αξιολόγηση HITL πρέπει επίσης να μετρά αν η ανθρώπινη ανατροφοδότηση χρησιμοποιείται αποτελεσματικά.

### Καμπύλες Μάθησης

Μια **καμπύλη μάθησης** σχεδιάζει την απόδοση μοντέλου συναρτήσει του αριθμού επισημασμένων παραδειγμάτων. Μια απότομη καμπύλη μάθησης (ταχεία βελτίωση με λίγες ετικέτες) δείχνει ότι η στρατηγική σχολιασμού επιλέγει ενημερωτικά παραδείγματα. Μια επίπεδη καμπύλη μάθησης δείχνει ότι η πρόσθετη επισήμανση παρέχει φθίνουσες αποδόσεις.

```{code-cell} python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_classification
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import f1_score, roc_auc_score
from sklearn.model_selection import StratifiedShuffleSplit

rng = np.random.default_rng(42)
X, y = make_classification(n_samples=5000, n_features=30, n_informative=15,
                            weights=[0.8, 0.2], random_state=42)
X_test, y_test = X[4000:], y[4000:]
X_pool, y_pool = X[:4000], y[:4000]

label_sizes = [20, 40, 60, 100, 150, 200, 300, 400, 600, 800, 1000, 1500, 2000]
metrics = {'f1': [], 'auc': []}

for n in label_sizes:
    idx = rng.choice(len(X_pool), n, replace=False)
    clf = LogisticRegression(max_iter=500, class_weight='balanced')
    clf.fit(X_pool[idx], y_pool[idx])
    preds = clf.predict(X_test)
    probs = clf.predict_proba(X_test)[:, 1]
    metrics['f1'].append(f1_score(y_test, preds))
    metrics['auc'].append(roc_auc_score(y_test, probs))

# Fit learning curve: performance ≈ a - b/sqrt(n)
from scipy.optimize import curve_fit

def learning_curve_fn(n, a, b):
    return a - b / np.sqrt(n)

popt_f1, _ = curve_fit(learning_curve_fn, label_sizes, metrics['f1'], p0=[0.9, 2])
popt_auc, _ = curve_fit(learning_curve_fn, label_sizes, metrics['auc'], p0=[0.95, 1])

n_smooth = np.linspace(20, 3000, 200)

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(11, 4))

ax1.scatter(label_sizes, metrics['f1'], color='#2b3a8f', zorder=5, s=40)
ax1.plot(n_smooth, learning_curve_fn(n_smooth, *popt_f1), '--', color='#e05c5c',
         label=f'Fit: {popt_f1[0]:.3f} - {popt_f1[1]:.1f}/√n')
ax1.set_xlabel("Labeled examples"); ax1.set_ylabel("F1 score")
ax1.set_title("Learning Curve: F1"); ax1.legend(); ax1.grid(alpha=0.3)

ax2.scatter(label_sizes, metrics['auc'], color='#0d9e8e', zorder=5, s=40)
ax2.plot(n_smooth, learning_curve_fn(n_smooth, *popt_auc), '--', color='#e05c5c',
         label=f'Fit: {popt_auc[0]:.3f} - {popt_auc[1]:.2f}/√n')
ax2.set_xlabel("Labeled examples"); ax2.set_ylabel("AUROC")
ax2.set_title("Learning Curve: AUROC"); ax2.legend(); ax2.grid(alpha=0.3)

plt.tight_layout()
plt.savefig('learning_curves.png', dpi=150)
plt.show()

# Estimate the annotation budget needed to reach a target performance
target_f1 = 0.80
n_needed = (popt_f1[1] / (popt_f1[0] - target_f1)) ** 2
print(f"Estimated labels needed to reach F1={target_f1}: {n_needed:.0f}")
```

### Ανάλυση Απόδοσης Επένδυσης (ROI)

Η ROI ανθρώπινης ανατροφοδότησης απαντά: για κάθε πρόσθετη ετικέτα, πόσο βελτιώνεται η απόδοση μοντέλου;

$$
\text{ROI}(n) = \frac{\Delta \text{performance}(n)}{\text{cost per label}}
$$

Καθώς ένα μοντέλο ωριμάζει (και καθώς εξαντλούνται τα εύκολα παραδείγματα μάθησης), η ROI συνήθως μειώνεται. Η πρακτική συνέπεια: οι προϋπολογισμοί σχολιασμού πρέπει να είναι μεγαλύτεροι στα πρώτα στάδια, με περισσότερες ετικέτες που συλλέγονται στα αρχικά στάδια όταν η ROI είναι υψηλότερη.

## Ανθρώπινη Αξιολόγηση

Για γενεστικά συστήματα και υποκειμενικές εργασίες, η ανθρώπινη αξιολόγηση παραμένει το χρυσό πρότυπο.

### Άμεση Αξιολόγηση (DA)

Οι σχολιαστές βαθμολογούν αποτελέσματα σε απόλυτη κλίμακα (π.χ. 1–100 για ποιότητα μετάφρασης, ή 1–5 για χρησιμότητα απάντησης). Η DA έχει τυποποιηθεί στην αξιολόγηση μηχανικής μετάφρασης (benchmarks WMT).

**Βέλτιστες πρακτικές για DA:**
- Τυχαιοποιείτε τη σειρά των αποτελεσμάτων για να αποτρέψετε την αγκύρωση
- Χρησιμοποιείτε επαρκή αριθμό σχολιαστών ανά στοιχείο (ελάχιστο 3–5)
- Συμπεριλαμβάνετε ελέγχους ποιότητας (προφανώς καλά και κακά παραδείγματα για εντοπισμό αδύνατων αξιολογητών)
- Αναφέρετε συμφωνία μεταξύ σχολιαστών παράλληλα με συγκεντρωτικές βαθμολογίες

### Συγκριτική Αξιολόγηση

Οι σχολιαστές επιλέγουν μεταξύ δύο αποτελεσμάτων: "Ποιο είναι καλύτερο;" Οι συγκριτικές κρίσεις είναι ταχύτερες και πιο συνεπείς από τις απόλυτες βαθμολογίες (βλ. Κεφάλαιο 8). **Τα συστήματα βαθμολόγησης ELO** (δανεισμένα από το σκάκι) μετατρέπουν τα αποτελέσματα ζευγαρωτών συγκρίσεων σε μια συνεχή κατάταξη ποιότητας.

```{code-cell} python
import numpy as np

def update_elo(rating_a, rating_b, outcome_a, k=32):
    """Update ELO ratings. outcome_a: 1=A wins, 0=B wins, 0.5=tie."""
    expected_a = 1 / (1 + 10 ** ((rating_b - rating_a) / 400))
    expected_b = 1 - expected_a
    new_a = rating_a + k * (outcome_a - expected_a)
    new_b = rating_b + k * ((1 - outcome_a) - expected_b)
    return new_a, new_b

# Simulate 5 model versions being compared pairwise
rng = np.random.default_rng(42)
true_quality = [0.60, 0.70, 0.75, 0.80, 0.85]  # underlying model quality
n_models = len(true_quality)
elo_ratings = {i: 1000.0 for i in range(n_models)}

for _ in range(500):  # 500 pairwise comparisons
    i, j = rng.choice(n_models, 2, replace=False)
    p_i_wins = true_quality[i] / (true_quality[i] + true_quality[j])
    outcome = 1.0 if rng.random() < p_i_wins else 0.0
    elo_ratings[i], elo_ratings[j] = update_elo(elo_ratings[i], elo_ratings[j], outcome)

print("ELO Rankings after 500 comparisons:")
sorted_models = sorted(elo_ratings.items(), key=lambda x: x[1], reverse=True)
for rank, (model_id, elo) in enumerate(sorted_models, 1):
    print(f"  Rank {rank}: Model {model_id}  ELO={elo:.1f}  True quality={true_quality[model_id]:.2f}")
```

### Συμπεριφορικές Δοκιμές (CheckList)

Το **CheckList** {cite}`ribeiro2020beyond` είναι μια μεθοδολογία για συστηματική συμπεριφορική αξιολόγηση μοντέλων NLP. Αντί για τυχαία σύνολα ελέγχου, σχεδιάζει περιπτώσεις δοκιμής που ανιχνεύουν συγκεκριμένες δυνατότητες:

- **Δοκιμές Ελάχιστης Λειτουργικότητας (MFT):** Χειρίζεται το μοντέλο απλές, προφανείς περιπτώσεις;
- **Δοκιμές Αμεταβλητότητας (INV):** Αλλάζει η έξοδος του μοντέλου όταν δεν πρέπει (π.χ., κατά την παράφραση);
- **Δοκιμές Κατευθυντικής Προσδοκίας (DIR):** Αλλάζει η έξοδος του μοντέλου στην αναμενόμενη κατεύθυνση όταν αλλάζει η είσοδος;

Το CheckList κάνει την ανθρώπινη αξιολόγηση στοχευμένη και εφαρμόσιμη: αντί για έναν μοναδικό αριθμό ακρίβειας, παρέχει ένα προφίλ δυνατοτήτων.

## Μέτρηση Ευθυγράμμισης με Ανθρώπινη Πρόθεση

Για συστήματα RLHF, η μέτρηση ευθυγράμμισης είναι μια κεντρική πρόκληση αξιολόγησης.

**Αξιολόγηση μοντέλου ανταμοιβής:** Η ακρίβεια του μοντέλου ανταμοιβής σε ένα κρατημένο σύνολο ελέγχου προτίμησης. Οι Ouyang et al. {cite}`ouyang2022training` αναφέρουν περίπου 72% ζευγαρωτή ακρίβεια για το μοντέλο ανταμοιβής του InstructGPT· ως κατά προσέγγιση σημείο αναφοράς, αριθμοί σε αυτή την περιοχή αναφέρονται συνήθως για παρόμοιες γραμμές RLHF, αν και τα αποτελέσματα ποικίλλουν ευρέως ανά εργασία και ποιότητα δεδομένων.

**Ποσοστό νίκης:** Δεδομένων δύο εκδόσεων μοντέλου (π.χ. βάση SFT έναντι RLHF fine-tuned), ποιο κλάσμα απαντήσεων κερδίζει το μοντέλο RLHF σε ανθρώπινες ζευγαρωτές συγκρίσεις;

**GPT-4 ως αξιολογητής:** Η χρήση ικανού LLM για αξιολόγηση απαντήσεων έχει γίνει κοινή για ταχεία επανάληψη. Οι Gilardi et al. {cite}`gilardi2023chatgpt` και Zheng et al. {cite}`zheng2023judging` βρίσκουν συμφωνία αξιολογητή LLM με ανθρώπινη κρίση περίπου 0,7 έως 0,9 ανάλογα με την εργασία — χρήσιμο για ταχεία σύγκριση A/B, αλλά λιγότερο αξιόπιστο για εντοπισμό κολακείας, πολιτισμικής απόχρωσης ή ζητημάτων ασφάλειας.

**Εντοπισμός κολακείας:** Μετρήστε αν το μοντέλο αλλάζει τις απαντήσεις του βάσει υπονοούμενης προτίμησης χρήστη (π.χ. "Νομίζω ότι το Χ είναι σωστό· τι νομίζεις;"). Ένα καλά ευθυγραμμισμένο μοντέλο δεν πρέπει να είναι κολακευτικό.

## Δοκιμές A/B σε Αναπτυγμένα Συστήματα

Για συστήματα σε παραγωγή, η τελική αξιολόγηση είναι οι **δοκιμές A/B**: δρομολογήστε ένα κλάσμα χρηστών στη νέα έκδοση μοντέλου και μετρήστε τα αποτελέσματα κατάντη.

Οι δοκιμές A/B δίνουν μια αμερόληπτη εκτίμηση ποιότητας μοντέλου στο πραγματικό περιβάλλον ανάπτυξης, αποτυπώνοντας επιδράσεις που η εργαστηριακή αξιολόγηση χάνει (συμπεριφορά χρήστη, κατανομή πληθυσμού, οριακές περιπτώσεις).

Η πρόκληση: κατάλληλες μετρικές κατάντη. Οι μετρικές αφοσίωσης (κλικ, διάρκεια συνεδρίας) μπορεί να ανταμείψουν χειριστική συμπεριφορά. Τα ποσοστά ολοκλήρωσης εργασιών ή οι έρευνες ικανοποίησης χρηστών είναι καλύτερα ευθυγραμμισμένα αλλά θορυβωδέστερα.
