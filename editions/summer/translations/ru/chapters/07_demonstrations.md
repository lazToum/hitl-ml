# Обучение на основе демонстраций

Когда задачу трудно формализовать, но легко продемонстрировать, обучение на примерах может быть эффективнее, чем определение через правила. Эксперт-человек показывает роботизированному манипулятору, как захватить объект; программист взаимодействует с IDE, предоставляя последовательность правильных правок; шахматный гроссмейстер разыгрывает партию. **Обучение на основе демонстраций** извлекает политику из таких поведенческих данных, избавляя от необходимости ручного конструирования функции вознаграждения или явной спецификации задачи.

---

## Клонирование поведения

Простейший подход — **клонирование поведения (BC)**: рассматривать демонстрацию как данные с учителем и обучать отображение из состояний в действия.

Дан датасет пар состояние–действие $\mathcal{D} = \{(s_i, a_i)\}$ от эксперта-демонстратора, обучается политика $\pi_\theta(a \mid s)$ путём минимизации отрицательного логарифма правдоподобия:

$$
\mathcal{L}_\text{BC}(\theta) = -\frac{1}{|\mathcal{D}|} \sum_{(s, a) \in \mathcal{D}} \log \pi_\theta(a \mid s)
$$

Это в точности стандартное обучение с учителем, применённое к последовательным данным.

### Проблема ковариатного сдвига

BC имеет фундаментальный изъян: **сдвиг распределения** между обучением и развёртыванием. Демонстрации эксперта охватывают состояния, посещаемые экспертом. Однако в процессе развёртывания обученная политика может принимать несколько иные решения, попадая в состояния, которые эксперт никогда не посещал, — состояния, где у политики нет надзора и где она может давать серьёзный сбой.

Ошибки **накапливаются**: небольшое отклонение приводит к незнакомому состоянию, где чуть неверное действие приводит к ещё более незнакомому состоянию, и так далее. Производительность деградирует как $O(T^2 \epsilon)$, где $T$ — длина эпизода, а $\epsilon$ — частота ошибок на каждом шаге, — что значительно хуже деградации $O(T\epsilon)$ оракульной политики {cite}`ross2010efficient`.

```{admonition} Пример: автономное вождение
:class: note

Модель клонирования поведения для удержания полосы, обученная на данных вождения человека, хорошо работает на прямых дорогах (состояния близко к обучающему распределению). Но как только автомобиль чуть отклонится — в состояние, в котором ни один водитель не оказался бы, поскольку уже скорректировал бы курс, — модель не имеет данных для руководства и может съехать с дороги.
```

```text
Алгоритм DAgger:
  Инициализация: D <- {} (пустой датасет)
  Обучить начальную политику pi_1 на M демонстрациях эксперта

  для итерации i = 1, 2, ..., N:
    1. Запустить pi_i в среде для сбора состояний {s_1, ..., s_t}
    2. Запросить у эксперта действия для каждого посещённого состояния: a_t = pi*(s_t)
    3. Агрегировать: D <- D u {(s_1, a_1), ..., (s_t, a_t)}
    4. Обучить pi_{i+1} методом обучения с учителем на D
```

DAgger достигает сожаления $O(T\epsilon)$ — такого же, как у оракульной политики, — поскольку обучающее распределение сходится к совпадению с распределением развёртывания.

Ключевое требование: эксперт должен быть доступен для запросов в любом состоянии, включая состояния, которые эксперт никогда бы не посетил естественным образом. Это осуществимо в симуляции (попросить эксперта скорректировать робота из нестандартной конфигурации), но может быть затруднительным или небезопасным в реальных физических системах.

## Обратное обучение с подкреплением

Иногда поведение эксперта лучше понимать не как последовательность действий для имитации, а как результат оптимизации неизвестной функции вознаграждения. **Обратное обучение с подкреплением (IRL)** {cite}`ng2000algorithms` восстанавливает эту латентную функцию вознаграждения из демонстраций.

Дана демонстрация $\tau = \{(s_1, a_1), \ldots, (s_T, a_T)\}$, IRL находит функцию вознаграждения $R(s, a)$ такую, что политика эксперта оптимальна при $R$.

Преимущество IRL перед BC: если нам удаётся восстановить истинную функцию вознаграждения, мы можем повторно оптимизировать её в новых средах, с другой динамикой или с улучшенными планировщиками — обобщаясь далеко за пределы продемонстрированных сценариев.

### MaxEnt IRL

**MaxEnt IRL** {cite}`ziebart2008maximum` разрешает проблему неоднозначности IRL (существует множество функций вознаграждения, согласованных с любым набором демонстраций) путём выбора функции вознаграждения, которая, будучи согласованной с продемонстрированным поведением, приводит к распределению по траекториям с *максимальной энтропией*. Траектории распределены как:

$$
P(\tau \mid R) \propto \exp\left(\sum_t R(s_t, a_t)\right)
$$

Цель обучения состоит в согласовании ожидаемых характеристик эксперта $\mu_E = \mathbb{E}_{\tau \sim \pi^*}[\phi(\tau)]$ с ожидаемыми характеристиками модели $\mu_\theta = \mathbb{E}_{\tau \sim \pi_\theta}[\phi(\tau)]$.

---

## GAIL: генеративное состязательное имитационное обучение

**GAIL** {cite}`ho2016generative` полностью обходит обучение функции вознаграждения, используя GAN-подобную формулировку для непосредственного согласования распределения состояний и действий с экспертом.

Дискриминатор $D_\psi$ обучается различать пары состояние–действие эксперта $(s, a) \sim \pi^*$ и пары политики $(s, a) \sim \pi_\theta$:

$$
\mathcal{L}_D = -\mathbb{E}_{\pi^*}[\log D_\psi(s,a)] - \mathbb{E}_{\pi_\theta}[\log(1 - D_\psi(s,a))]
$$

Генератор (политика $\pi_\theta$) обучается обманывать дискриминатор — то есть производить пары состояние–действие, похожие на экспертские. Выход дискриминатора $\log D_\psi(s,a)$ служит сигналом вознаграждения для политики.

GAIL достигает производительности уровня эксперта на бенчмарках непрерывного управления, используя значительно меньше демонстраций, чем BC, и обобщается лучше, чем MaxEnt IRL в сложных средах.

## Клонирование поведения в NLP: практический пример

```{code-cell} python
import numpy as np
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

torch.manual_seed(42)

# -----------------------------------------------
# Toy NLP task: rewriting sentences to be more formal
# We simulate this as a simple sequence transformation
# In practice: fine-tuning a seq2seq model on expert rewrites
# -----------------------------------------------

class SimpleSeq2Seq(nn.Module):
    def __init__(self, vocab_size=100, embed_dim=32, hidden_dim=64):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.encoder = nn.GRU(embed_dim, hidden_dim, batch_first=True)
        self.decoder = nn.GRU(embed_dim, hidden_dim, batch_first=True)
        self.proj = nn.Linear(hidden_dim, vocab_size)
        self.hidden_dim = hidden_dim

    def forward(self, src, tgt):
        src_emb = self.embed(src)
        _, hidden = self.encoder(src_emb)
        tgt_emb = self.embed(tgt)
        out, _ = self.decoder(tgt_emb, hidden)
        return self.proj(out)

# Generate synthetic demonstration data
VOCAB = 100
rng = np.random.default_rng(42)
N, SEQ_LEN = 1000, 12

src_seqs = torch.tensor(rng.integers(1, VOCAB, (N, SEQ_LEN)), dtype=torch.long)
# "Expert" transformation: shift tokens by 1 (toy formalization)
tgt_seqs = torch.clamp(src_seqs + 1, 1, VOCAB - 1)
tgt_in  = torch.cat([torch.ones(N, 1, dtype=torch.long), tgt_seqs[:, :-1]], dim=1)

dataset = TensorDataset(src_seqs, tgt_in, tgt_seqs)
loader = DataLoader(dataset, batch_size=64, shuffle=True)

model = SimpleSeq2Seq(vocab_size=VOCAB)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.CrossEntropyLoss(ignore_index=0)

# Behavioral cloning training
train_losses = []
for epoch in range(20):
    epoch_loss = 0
    for src, tgt_i, tgt_o in loader:
        logits = model(src, tgt_i)
        loss = criterion(logits.reshape(-1, VOCAB), tgt_o.reshape(-1))
        optimizer.zero_grad(); loss.backward(); optimizer.step()
        epoch_loss += loss.item()
    train_losses.append(epoch_loss / len(loader))

print(f"Initial loss: {train_losses[0]:.3f}")
print(f"Final loss:   {train_losses[-1]:.3f}")

# Evaluate: check token accuracy on held-out examples
model.eval()
with torch.no_grad():
    src_test = src_seqs[-100:]
    tgt_test_in = tgt_in[-100:]
    tgt_test_out = tgt_seqs[-100:]
    logits = model(src_test, tgt_test_in)
    preds = logits.argmax(dim=-1)
    acc = (preds == tgt_test_out).float().mean().item()
    print(f"Token accuracy on held-out set: {acc:.3f}")
```

## Сравнение методов имитационного обучения

| Метод            | Требует вознаграждения? | Эксперт опрашивается онлайн? | Обобщается на новую динамику? | Сложность   |
|------------------|------------------------|------------------------------|-------------------------------|-------------|
| Клонирование BC  | Нет                    | Нет                          | Плохо (сдвиг распределения)   | Низкая      |
| DAgger           | Нет                    | Да                           | Умеренно                      | Средняя     |

| GAIL             | Нет                    | Нет                          | Хорошо                        | Высокая     |

## Приложения

**Робототехника.** Обучение роботов манипулировать объектами, ориентироваться в среде или выполнять бытовые задачи. Физические демонстрации собираются через телеуправление или кинестетическое обучение.

**Автономное вождение.** Ранние системы автономного вождения, такие как ALVINN {cite}`pomerleau1989alvinn` и DAVE от NVIDIA, в значительной мере опирались на клонирование поведения на основе данных вождения людьми.

**Игровой ИИ.** Имитационное обучение на игровом процессе людей обеспечивает начальный запуск агентов перед дообучением с помощью RL. AlphaStar обучался на репликах людей перед применением RL; этот подход распространён, когда доступны демонстрации уровня человека.

**Генерация кода.** Дообучение языковых моделей на высококачественных демонстрациях кода (GitHub Copilot, Codex) является формой клонирования поведения.

**Клиническая поддержка принятия решений.** Обучение на последовательностях решений экспертов-врачей для сложных протоколов.
