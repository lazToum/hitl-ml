# 何谓人机协同机器学习？

```{epigraph}
一台能从经验中学习的机器……但前提是你给它正确的经验。
-- 改述自艾伦·图灵
```

## 自动化悖论

每一次自动化浪潮都会对人类注意力产生新的需求。当航空公司引入自动驾驶仪时，飞行员变成了监控者——他们的职责不再是对飞机进行时刻到时刻的操控，而是承担更艰难的任务：判断*何时*应该接管。当超市引入自助结账时，人们发现这些系统每笔交易所需的人工监督实际上比传统收银员更多，而非更少。当机器学习开始在医疗、金融、招聘和内容审核等领域大规模做出决策时，它创造了对人类判断力的巨大且持续的需求。

这就是**自动化悖论** {cite}`bainbridge1983ironies`：一个自动化系统越有能力，其失败的后果就越严重，因此对稳健的人类监督的需求也就越强烈。机器学习也不例外。

人在回路机器学习（HITL ML）是一个认真对待这一悖论、并将其从一开始就融入系统设计的领域。HITL ML不是将人类参与视为最终将被移除的临时脚手架，而是将人机交互视为一种*特性*——一种能使模型更准确、更符合人类价值观且更值得信赖的信号源、纠错机制和引导力量。

## 人在回路机器学习的定义

文献中对HITL ML没有统一认可的定义，该术语以几种相互交叠的方式被使用。本手册采用一个宽泛而精确的定义：

:::{admonition} 定义
:class: important

**人在回路机器学习**是指任何以*有意识、有结构且持续进行*的方式将人类反馈纳入学习过程的机器学习系统或方法论——不仅仅在数据集创建时，而是贯穿训练、评估和部署的全过程。

:::

该定义包含三个关键条款：

**有意识。** HITL并非无意为之的人类影响（例如，训练数据最初由人类创建这一事实）。它指的是被明确设计为主动征求、融合并从人类反馈中受益的系统。

**有结构。** 反馈具有明确的形式——一个标签、一次纠正、一个偏好判断、一次示范——以及在学习算法中的明确角色。

**持续进行。** 反馈回路随时间推移持续运转，使系统能够在遭遇新情境、犯下错误并接受人类引导的过程中不断改进。

该定义涵盖了经典标注流程、主动学习、交互式机器学习、RLHF和模仿学习。它排除了被动数据收集和纯离线监督学习（尽管界限较为模糊，我们将在第2章中讨论）。

## 简史

### 专家系统与知识工程（1960年代至1980年代）

最早的人工智能系统几乎完全是人在回路的：知识工程师与领域专家共同工作数月，将规则艰辛地编码进MYCIN和DENDRAL等专家系统中。每一条知识都由人类明确提供。机器是执行者；人类是知识的源泉。

这些系统在狭窄领域内的表现出人意料地好，但十分脆弱——无法超越手工制定的规则进行泛化，且维护成本高昂。

### 统计学转向（1980年代至2000年代）

1980年代和1990年代转向统计机器学习改变了人类参与的性质。人类不再以规则的形式编码知识，而是提供*示例*——允许模型从中推断规律的标注数据集。人类的角色变成了标注者：为文档标记类别、为图像打标签、转录语音。

这是一大进步，但它造成了新的瓶颈：**标注数据代价高昂**。研究者开始思考如何最高效地利用人类标注资源。这个问题催生了**主动学习**，它在1990年代初被正式提出 {cite}`cohn1994improving`。

### 深度学习时代（2010年代至今）

深度学习革命创造了一种新的格局：具有数十亿参数的模型能够从数据中学习极其复杂的函数——但需要相应规模的庞大标注数据集。ImageNet（1400万张标注图像）及其后的大规模标注项目既展示了规模化的力量，也揭示了其成本。

与此同时，大规模部署机器学习暴露了新的问题：模型在平均水平上准确，但对特定群体系统性地出错；模型自信地产生幻觉事实；模型针对可测量的代理目标而非人类价值观进行优化。这些失败促使人们寻求新形式的人类参与：不仅仅是标注，而是*对齐*——使模型按照人类实际意图行事的工程。

这一对齐导向的HITL工作最清晰的体现是**基于人类反馈的强化学习（RLHF）** {cite}`christiano2017deep`，它成为了InstructGPT {cite}`ouyang2022training`等系统以及现代语言模型指令遵循能力的核心支柱。

## 为何选择HITL？人类判断力的价值

是什么使人类判断力有价值到值得被融入机器学习系统？以下几个属性值得关注：

### 1. 常识与世界知识

人类将大量的背景知识带入任何任务。当一位放射科医生为X光片做标注时，她调用的是多年的训练经验、对解剖学的理解，以及关于疾病表现形态的隐性知识——这些知识极难从数据中完整指定或获取。

### 2. 语义锚定

标签之所以有意义，是因为人类理解其含义。ImageNet中的"猫"类别指代一个人类凭直觉就能识别、但任何正式定义都无法完全捕捉的模糊概念。模型学习的是标签的外延（哪些图像映射到它），却可能未能学到概念本身，从而在任何人类都能正确处理的边缘案例上失败。

### 3. 价值对齐

人类具有机器学习模型无法单从数据中推导出来的偏好、价值观和伦理判断。一段文本是否"有帮助"或"有害"，并非纯粹的经验性问题——它取决于因个体和情境而异的人类价值观。HITL是将这些价值观传递给机器学习系统的主要机制。

### 4. 适应性

人类判断力能够适应新颖情境，而无需重新训练。在历史数据上训练的模型可能在世界发生变化时灾难性地失败；人类可以识别新颖性并做出恰当回应。

### 5. 问责性

在高风险领域——医疗、法律、金融——决策需要向人类负责。HITL系统通过将人类保持在能够理解、验证和覆盖机器行为的位置，使这种问责性变得可行。

## 反馈回路

HITL ML的核心结构是模型与一个或多个人类之间的反馈回路：

```text
+---------------------------------------------+
|                                             |
|   模型发出预测 / 请求                        |
|   ---------------------------------->       |
|                                   人类      |
|   人类提供反馈        <-------- |
|   ----------------------------------        |
|                                             |
|   模型在反馈基础上更新                       |
|                                             |
+---------------------------------------------+
```

在不同的HITL范式中，反馈的性质差异极大：

| 反馈类型       | 示例                                         | 主要范式               |
|----------------|----------------------------------------------|------------------------|
| 类别标签       | "这封邮件是垃圾邮件"                         | 监督学习               |
| 纠正           | "该实体应为ORG，而非PER"                     | 主动学习 / 交互式机器学习 |
| 偏好           | "回复A优于B"                                 | RLHF / 排序学习        |
| 示范           | 向机器人展示如何抓取物体                     | 模仿学习               |
| 自然语言       | "请让你的回答更简洁"                         | 指令微调               |
| 隐式信号       | 用户点击了 / 未点击                          | 隐式反馈               |

## HITL不是什么

有必要明确界定哪些情况*不在*我们定义的范围之内。

**HITL不等同于部署中的人在回路**（有时称为"人在环上"，human on the loop），在那种情况下，人类监控自动化决策并可以覆盖它们，但并未将纠正反馈回训练中。我们将在第2章讨论这一区别。

**HITL不等同于单独的弱监督。** 像Snorkel这样的程序化标注系统使用标注函数（通常是人类撰写的规则）在规模上生成噪声标签。这是一种结构化人类输入，但其反馈并非HITL通常意义上的迭代式反馈。

**HITL不等同于简单地使用标注数据。** 每个监督学习模型都使用人类标注数据。HITL特指人类反馈是学习过程的*主动、迭代*组成部分的系统。

## 人类反馈的经济学

人类反馈有价值，但代价高昂。在专科医师执行的情况下，医学影像标注的成本从数十美元到数百美元不等 {cite}`monarch2021human`。在Amazon Mechanical Turk等平台上，众包工人标签的成本可以低至每条0.01至0.10美元 {cite}`hara2018data`，但质量也低得多。HITL ML的根本挑战在于**最大化每单位人类反馈的价值**。

这引出了一个贯穿本手册大部分内容的统一问题：

:::{admonition} HITL ML的核心问题
:class: tip

*在人类时间和注意力预算固定的情况下，我们应该如何决定向人类提问什么、何时提问，以及如何将他们的回答融入模型训练？*

:::

这个问题的答案取决于领域、反馈形式、标注成本、错误风险以及模型当前状态——这正是为什么HITL ML是一个丰富且仍在演进的学科。
