# 人机协同在自然语言处理中的应用

自然语言是HITL ML产生最深远影响——同时也涌现出其最深层概念困难——的领域。语言本质上是社会性的：其意义由人类社群共同构建，其语用依赖于情境和意图，其质量只能由人类读者来判断。但这也意味着NLP标注不仅仅是收集观测数据的过程，它是一个*建构*类别的过程。

---

## NLP标注中的建构性问题

在医学影像中，存在客观真相：肿瘤要么存在，要么不存在。放射科医生的标签可能存在不确定性，但它试图追踪某种真实的东西。NLP标注往往根本不同。当标注者将一条推文标记为"有毒"时，推文中并不存在某种"毒性分子"需要我们去检测。**标签建构了类别本身。**

这具有深远影响，而这些影响经常被严重低估：

**标注者群体定义了这一现象。** "冒犯性言论"的标注方案，编码了设计它的人以及应用它的人的感知。一支来自单一人口背景的英语标注者团队，在企业政策团队撰写的指南下工作，产生的数据集反映的是该团队对冒犯的理解——而非某种普遍的人类标准。在此类数据上训练的模型将表现出相同的隐性边界。

**指南是理论，无论它们是否承认这一点。** 每个标注方案都在做本体论上的主张。决定"讽刺"和"挖苦"属于同一类别是一个理论主张，而非中立的便利之举。决定将"愤怒"标注为单一类别而不区分"正义之怒"和"敌意之怒"，抹去了一个对下游任务可能重要的区分。这些决定是在生产压力下做出的，鲜少被重新审视。

**标签的时间不稳定性。** 社会语言在演变。一个在2018年训练的毒性模型会对2024年的内容产生误分类，不是因为它在统计上欠训练，而是因为某些词语的社会意义已经发生了变化。NLP标注不是从静态分布中采样——它是从一个不断移动的目标中采样，而标注行为本身也在一定程度上参与构建这个目标。

:::{admonition} 标注伪像问题
:class: important

Geva等人 {cite}`geva2019annotator` 证明，自然语言推理（NLI）数据集包含由标注过程本身引入的系统性伪像。被要求为给定前提撰写"蕴含"假设的标注者，倾向于使用某些句法模式；撰写"矛盾"假设的标注者使用另一些模式。模型学会基于这些伪像进行分类，而非基于预期的语义关系——它们解决的是标注任务，而非底层的NLP任务。

这不是粗心大意。这是让人类构建符合标签的示例这一做法的固有后果。HITL过程插入了一个从未打算出现在数据中的系统性信号。
:::

## 文本分类标注

最简单的NLP标注任务是为一个文本文档分配类别。情感分析、主题分类、意图检测和垃圾邮件过滤都是分类任务。

**文本分类特有的挑战：**

*主观性。* "有毒"或"冒犯性"等类别本质上是主观的，因文化情境、标注者背景和平台情境而异。不同标注者对冒犯性的感知因人口背景而异显著——这一事实对公平性有直接影响 {cite}`blodgett2020language`。

*标签歧义。* 许多文档属于多个类别或处于边界上。一篇60%正面、40%负面的评论是真正模糊的，而非被错误标注。强制给出单一标签会丢弃真实信息（参见第13章关于软标签和标注者分歧的讨论）。

*标签粒度。* 二分类情感模型可能足以用于社交媒体监控；七分制量表可能是产品反馈分析所必需的。正确的粒度取决于下游任务，但通常在标注之前就已固定——这是一个在信息不足的情况下做出的重要设计决策。

## 命名实体识别

NER标注需要识别文本片段并分配实体类型（人名、组织机构名、地名等）。由于以下原因，标注比文档分类更为复杂：

**片段边界具有歧义性。** 在"苹果公司CEO蒂姆·库克宣布……"中，人名实体是"蒂姆·库克"还是"苹果公司CEO蒂姆·库克"？指南必须明确处理这类情况，标注者在片段边界上的一致性始终低于在类型分配上的一致性。

**类型分配需要世界知识。** "Apple（苹果）"在一种情境下是ORG，在另一种情境下是PRODUCT，在"apple pie（苹果派）"中两者都不是。标注者需要足够的领域知识来做出正确的类型分配。

**嵌套实体。** "加州大学伯克利分校"包含嵌套在LOCATION内的ORGANIZATION。标准BIO标注无法表示嵌套实体；需要更复杂的方案（例如BIOES或基于图的格式）。

**标注效率。** 用基线NER模型进行预标注，允许标注者纠正预测而非从零开始标注，可以显著提高标注速度。在一项临床NER研究中，观察到吞吐量提升了30至60% {cite}`lingren2014evaluating`；这类收益的大小在很大程度上取决于基线模型质量和领域。

## 关系抽取与语义标注

除了识别实体，关系抽取还需要标注实体之间的*关系*：

- 标注者必须理解实体和连接它们的谓词
- 关系类型具有复杂的语义区别（WORKS\_AT vs. EMPLOYED\_BY vs. FOUNDED）
- 负样本（没有关系的实体对）远多于正样本

**关系抽取的标注者间一致性**往往低于分类任务。对于定义明确的方案，$\kappa$ 值通常在0.65至0.80的范围内 {cite}`pustejovsky2012natural`；对于复杂的语义关系（事件因果、时序关系），一致性可能会明显更低，具体取决于方案设计和标注者培训。

## 机器翻译译后编辑（MTPE）

机器翻译译后编辑是HITL NLP的一种成熟形式。人工译者纠正机器翻译系统的输出，而非从头翻译：

**轻度译后编辑（LPE）：** 只纠正关键错误。适用于翻译质量要求中等的情况。

**全面译后编辑（FPE）：** 将输出纠正至发布质量。编辑后的输出通常成为进一步MT改进的训练数据——真正的人在回路精化循环。

**HTER（人工目标译文编辑率）：** 衡量MT输出与译后编辑翻译之间的编辑距离，以句子长度进行归一化 {cite}`graham2015accurate`。作为粗略的从业者经验法则，HTER低于约0.3通常被认为是良好的MT输出；高于0.5时，从头翻译可能更快——尽管这些阈值因领域和语言对而异。

## 对话式AI与对话标注

对话标注引入了时序结构：

- **轮次级标注：** 为每个轮次添加标签（意图、情感、质量）
- **对话级标注：** 评估整体连贯性、任务完成情况、用户满意度
- **交互轨迹标注：** 识别对话中的特定失败时刻

HITL在对话中尤为重要，因为模型失败往往是微妙且累积性的：第3轮中的事实错误可能直到第7轮才变得明显。追踪对话的人类标注者可以识别出自动化指标完全遗漏的这些长程失败模式。

## 程序化标注与弱监督

当标注数据稀缺时，**弱监督**允许使用多个嘈杂、重叠的标注函数在规模上生成概率标签。**Snorkel** {cite}`ratner2017snorkel` 是典型框架：

```{code-cell} python
import numpy as np
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import f1_score

rng = np.random.default_rng(42)

# -------------------------------------------------------
# Toy weak supervision: sentiment classification
# 5 labeling functions (LFs) with different coverage/accuracy
# Label: +1 (positive), -1 (negative), 0 (abstain)
# -------------------------------------------------------

N = 1000
true_labels = rng.choice([-1, 1], size=N)
X_features = np.column_stack([
    true_labels * 0.8 + rng.normal(0, 0.5, N),
    rng.normal(0, 1, N),
    rng.normal(0, 1, N),
])

def make_lf(accuracy, coverage, seed):
    rng_lf = np.random.default_rng(seed)
    votes = np.zeros(N, dtype=int)
    active = rng_lf.random(N) < coverage
    correct = rng_lf.random(N) < accuracy
    votes[active & correct]  = true_labels[active & correct]
    votes[active & ~correct] = -true_labels[active & ~correct]
    return votes

LFs = np.column_stack([
    make_lf(accuracy=0.85, coverage=0.60, seed=1),
    make_lf(accuracy=0.78, coverage=0.45, seed=2),
    make_lf(accuracy=0.70, coverage=0.80, seed=3),
    make_lf(accuracy=0.90, coverage=0.30, seed=4),
    make_lf(accuracy=0.60, coverage=0.90, seed=5),
])

def majority_vote(LF_matrix):
    labels = []
    for i in range(LF_matrix.shape[0]):
        votes = LF_matrix[i][LF_matrix[i] != 0]
        labels.append(0 if len(votes) == 0 else int(np.sign(votes.sum())))
    return np.array(labels)

mv_labels = majority_vote(LFs)
covered = mv_labels != 0

print(f"Coverage:                    {covered.mean():.1%}")
print(f"MV accuracy (covered):       {(mv_labels[covered] == true_labels[covered]).mean():.3f}")

X_train, y_train = X_features[covered], mv_labels[covered]
X_test,  y_test  = X_features[~covered], true_labels[~covered]

if len(X_train) > 10 and len(X_test) > 10:
    clf = LogisticRegression().fit(X_train, y_train)
    preds = clf.predict(X_test)
    print(f"F1 on uncovered test set:    {f1_score(y_test, preds):.3f}")
```

## 用于语言模型的RLHF：标注的现实

第6章从技术角度介绍了RLHF。从NLP角度来看，标注任务比外界看起来更为困难。

**标注者实际被要求做的事情**——比较两个模型输出并指出哪个"更好"——听起来很简单。在实践中，"更好"是一个未被充分定义的构念，标注者使用个人启发式方法来解决它。有些人重视流畅性；有些人重视事实准确性；有些人惩罚冗长。没有严格的指南，产生的偏好数据集反映的不是抽象意义上的人类价值观，而是所雇用标注者群体的特定解决策略。

**关键的标注维度包括：**

- *有帮助性：* 回复是否很好地回答了问题？它是否信息丰富、清晰且适当详细？
- *事实性：* 回复在事实上是否准确？这要求评估者具有领域知识——一个在规模上会产生严重质量问题的要求，因为通才标注者无法验证专业性主张。
- *无害性：* 回复是否避免了有毒、有偏见、有害或不适当的内容？这些判断需要详细的指南，因为"有害"高度依赖情境，并随文化、时间和平台情境的变化而变化。
- *校准性：* 回复是否表达了适当的不确定性？标注者系统性地偏好听起来自信的回复，这产生了对抗适当认识论谦逊的奖励信号。

标准之间的相互作用是复杂的：对危险问题最有帮助的回复可能也是最有害的。指南必须指定如何权衡相互竞争的标准——而这些权衡实际上是政策决策，而非标注决策。标注者群体在制定政策。

**规模集中了人口影响。** 大型模型的RLHF涉及相对较小的标注者群体（数百至数千人），做出数十亿的下游决策。那个群体的人口和文化偏见以一种在标注更分散的情况下不会发生的方式，规模化地传播到模型行为中。这是当前RLHF流程中讨论最少的结构性问题之一。

## 标注-模型反馈回路

在NLP中，比在任何其他领域都更明显，标注和模型开发过程随时间变得相互交织：

1. 标注者使用模型现有输出作为参考进行校准（通常是隐式的）。
2. 奖励模型学习标注者倾向于偏好什么。
3. 生成器被微调以产生获得高奖励的输出。
4. 这些输出影响了后续标注轮次中"好"的标准看起来是什么样的。

这个反馈回路并不天然是病态的——它正是允许RLHF收敛的机制——但这意味着模型的行为是由一个标注过程本身也在推动的移动目标所塑造的。区分模型学到的是因为它反映了人类偏好，还是因为它学会了模式匹配标注者启发式方法，在经验上是困难的。

没有完美的解决方案。当前的最佳实践是使用定期收集的保留偏好判断来监测漂移，并将标注指南版本视为实验变量。

## 生成式NLP模型的评估

与具有明确准确率指标的分类模型不同，评估生成质量需要人类判断：

| 评估方法 | 描述 | 成本 |
|---|---|---|
| 直接评估（DA） | 标注者在量表上评定质量 | 中等 |
| 比较判断 | 标注者比较两个输出 | 低 |
| MQM（多维质量指标） | 结构化错误分类体系 | 高 |
| RLHF偏好 | 用于训练的偏好标签 | 中等 |
| LLM作为评判者 | LLM对输出评分（与人类的相关性中等） | 极低 |
| BERTScore、BLEU | 自动指标（与人类判断的相关性不完美） | 极低 |

自动指标（MT的BLEU、摘要的ROUGE）成本低廉，但与人类质量判断的相关性不完美 {cite}`reiter2018structured`。LLM作为评判者的方法与人类标注者的一致性中等 {cite}`zheng2023judging`，越来越多地被用于快速迭代，但不应取代人类评估进行最终评估。对于具有真实风险的决策，人类评估仍然是必要的。
