## 1. Μαθηματικό Πλαίσιο για την Ανθρώπινη Παρέμβαση βάσει Αβεβαιότητας

Τα παραδείγματα του Κεφαλαίου 1 μπορούν να τυποποιηθούν χρησιμοποιώντας τη θεωρία ποσοτικοποίησης αβεβαιότητας και πλαίσια θεωρίας αποφάσεων.

### 1.1 Βασική Ταξινόμηση Αβεβαιότητας

Ακολουθώντας τους Kendall και Gal (2017), διακρίνουμε δύο θεμελιώδεις τύπους αβεβαιότητας στα συστήματα μηχανικής μάθησης:

**Τυχαία Αβεβαιότητα (Aleatoric Uncertainty - Αβεβαιότητα Δεδομένων)**
- Αποτυπώνει τον εγγενή θόρυβο στις παρατηρήσεις.
- Δεν μπορεί να μειωθεί με τη συλλογή περισσότερων δεδομένων.
- Μαθηματικά: σ²_aleatoric = E[||y - f(x)||²]

**Γνωσιακή Αβεβαιότητα (Epistemic Uncertainty - Αβεβαιότητα Μοντέλου)**
- Αποτυπώνει την αβεβαιότητα σχετικά με τις παραμέτρους του μοντέλου.
- Μπορεί να μειωθεί με περισσότερα δεδομένα εκπαίδευσης.
- Μαθηματικά: σ²_epistemic = E[||f(x) - E[f(x)]||²]

### 1.2 Πλαίσιο Θεωρίας Αποφάσεων για την Ανθρώπινη Παρέμβαση

Έστω H μια ενέργεια ανθρώπινης παρέμβασης και M μια ενέργεια μόνο από τη μηχανή. Ο βέλτιστος κανόνας απόφασης είναι:

```
Action = {
    H  αν  EU(H) > EU(M)
    M  αν  EU(M) ≥ EU(H)
}
```

Όπου EU(·) αντιπροσωπεύει την αναμενόμενη ωφέλεια (expected utility) ενσωματώνοντας:
- **Κόστος ακρίβειας**: C_error × P(error|action)
- **Ανθρώπινο κόστος**: C_human × I(action = H)
- **Κόστος χρόνου**: C_time × T(action)

### 1.3 Βελτιστοποίηση Κατωφλίου Εμπιστοσύνης

Το παράδειγμα του Netflix «Συνεχίζετε την παρακολούθηση;» καταδεικνύει τη βέλτιστη επιλογή κατωφλίου. Έστω:
- p = πιθανότητα ο χρήστης να παρακολουθεί ακόμα ενεργά
- C_stream = κόστος περιττού streaming
- C_interrupt = κόστος διακοπής ενός ενεργού χρήστη

Βέλτιστο κατώφλι παρέμβασης: p* = C_interrupt / (C_interrupt + C_stream)

### 1.4 Οι Τρεις Τρόποι Αποτυχίας

Βασιζόμενοι στη συζήτηση του Κεφαλαίου 1, μπορούμε να χαρακτηρίσουμε τυπικά τρεις τρόπους αποτυχίας:

**Υπερβολική Ερώτηση (Κόπωση από Ειδοποιήσεις - Alert Fatigue):**
```
Αποτυχία όταν: threshold_ask << p*
Αποτέλεσμα: Υπερβολικές παρεμβάσεις, ο χρήστης αγνοεί το σύστημα
Μέτρηση: Υψηλό ποσοστό παρέμβασης, χαμηλή ακρίβεια παρέμβασης
```

**Ανεπαρκής Ερώτηση (Επικίνδυνη Υπερβολική Αυτοπεποίθηση):**
```
Αποτυχία όταν: threshold_ask >> p*
Αποτέλεσμα: Αυτόνομη δράση παρά την υψηλή αβεβαιότητα
Μέτρηση: Υψηλό ποσοστό σφάλματος στις αυτόνομες αποφάσεις
```

**Ερώτηση με Κακό Τρόπο (Το Πρόβλημα του Nest):**
```
Αποτυχία όταν: Το σύστημα ανιχνεύει αβεβαιότητα αλλά ενεργεί χωρίς επικοινωνία
Αποτέλεσμα: Σιωπηλά σφάλματα, ο χρήστης χάνει την εμπιστοσύνη στα «έξυπνα» χαρακτηριστικά
Μέτρηση: Υψηλό ποσοστό απενεργοποίησης χαρακτηριστικού, χαμηλή ικανοποίηση χρήστη
```

## 2. Μέθοδοι Εκτίμησης Αβεβαιότητας στην Πράξη

### 2.1 Bayesian Νευρωνικά Δίκτυα (BNNs)

Για την εκτίμηση της γνωσιακής αβεβαιότητας, τα συστήματα μπορούν να εφαρμόσουν μεταβολική συμπερασματολογία (variational inference):

**Προσέγγιση Μεταβολικής Υστεροπιθανότητας (Variational Posterior Approximation):**
```
q_θ(w) ≈ p(w|D)
όπου θ = {μ, σ} είναι οι μεταβολικές παράμετροι
```

**Προβλεπτική Αβεβαιότητα:**
```
p(y*|x*, D) = ∫ p(y*|x*, w) q_θ(w) dw
```

Προσέγγιση Monte Carlo μέσω εγκατάλειψης (dropout) (Gal & Ghahramani, 2016):
```python
def predict_with_uncertainty(model, x, n_samples=100):
    predictions = []
    for _ in range(n_samples):
        model.train()  # Ενεργοποίηση dropout
        pred = model(x)
        predictions.append(pred)
    
    mean = torch.mean(torch.stack(predictions), dim=0)
    uncertainty = torch.std(torch.stack(predictions), dim=0)
    return mean, uncertainty
```

### 2.2 Μέθοδοι Συνόλου (Ensemble Methods)

Εναλλακτική προσέγγιση για την ποσοτικοποίηση της αβεβαιότητας:

**Βαθιά Σύνολα (Deep Ensembles - Lakshminarayanan et al., 2017):**
```
Πρόβλεψη συνόλου: μ_ensemble = (1/M) Σ μ_i
Προβλεπτική διακύμανση: σ²_ensemble = (1/M) Σ [σ²_i + μ²_i] - μ²_ensemble
```

**Πλεονεκτήματα:**
- Δεν απαιτούνται αρχιτεκτονικές αλλαγές.
- Καλύτερη βαθμονόμηση (calibration) από τα μεμονωμένα μοντέλα.
- Αποτυπώνουν τόσο την τυχαία όσο και τη γνωσιακή αβεβαιότητα.

### 2.3 Αξιολόγηση Βαθμονόμησης

Κρίσιμη για τον καθορισμό του πότε θα κληθεί η ανθρώπινη παρέμβαση. Η κακή βαθμονόμηση μπορεί να οδηγήσει σε:
- **Υπερβολική εξάρτηση**: Χαμηλή αναφερόμενη αβεβαιότητα για λανθασμένες προβλέψεις.
- **Ανεπαρκή εξάρτηση**: Υψηλή αναφερόμενη αβεβαιότητα για σωστές προβλέψεις.

**Κατασκευή Διαγράμματος Αξιοπιστίας (Reliability Diagram):**
1. Ομαδοποίηση (binning) προβλέψεων ανά επίπεδο εμπιστοσύνης.
2. Υπολογισμός ακρίβειας σε κάθε ομάδα.
3. Σχεδίαση εμπιστοσύνης έναντι ακρίβειας.
4. Τέλεια βαθμονόμηση: διαγώνια γραμμή.

**Αναμενόμενο Σφάλμα Βαθμονόμησης (Expected Calibration Error - ECE):**
```
ECE = Σ_{m=1}^{M} (n_m/n) |acc(m) - conf(m)|
```
Όπου $m \in \{1, \ldots, M\}$ δείκτης $M$ ομάδων εμπιστοσύνης (συνήθως $M=10$ ομάδες ίσου εύρους στο $[0,1]$), $n_m$ ο αριθμός δειγμάτων στην ομάδα $m$, $n$ το συνολικό πλήθος δειγμάτων, $\text{acc}(m)$ το κλάσμα των σωστών προβλέψεων στην ομάδα $m$, και $\text{conf}(m)$ η μέση προβλεπόμενη εμπιστοσύνη στην ομάδα $m$.

## 3. Προκλήσεις Υλοποίησης στον Πραγματικό Κόσμο

### 3.1 Στρατηγικές Ερωτημάτων Ενεργού Μάθησης

Τα παραδείγματα «ερώτησης για βοήθεια» από το Κεφάλαιο 1 σχετίζονται με τη βιβλιογραφία της ενεργού μάθησης (Active Learning - Settles, 2009).

**Δειγματοληψία Αβεβαιότητας (Uncertainty Sampling):**
```
x* = argmax H(p(y|x))  # Μέγιστη εντροπία
```

**Ερώτηση μέσω Επιτροπής (Query by Committee):**
```
x* = argmax Disagreement(Committee, x)
```

**Αναμενόμενη Μείωση Σφάλματος (Expected Error Reduction):**
```
x* = argmin Σ p(y|x) Loss_expected(model_new, y)
```

### 3.2 Μάθηση με Επίγνωση Κόστους (Cost-Sensitive Learning)

Τα πραγματικά συστήματα πρέπει να εξισορροπούν πολλαπλά κόστη:

**Βελτιστοποίηση Πολλαπλών Στόχων:**
```
ελαχιστοποίηση α·Error_rate + β·Human_cost + γ·Latency
υπό τον περιορισμό: Safety_constraints
```

**Παράδειγμα: Σύστημα Ιατρικής ΤΝ**
- Error_rate: Ποσοστό ψευδώς αρνητικών για κρίσιμες καταστάσεις.
- Human_cost: Χρόνος εξέτασης από ακτινολόγο.
- Latency: Χρόνος μέχρι τη διάγνωση.
- Safety_constraint: Ποσοστό ψευδώς αρνητικών < ε.

### 3.3 Σχεδιαστικές Παράμετροι Διεπαφής

**Πληροφοριοθεωρητική Προσέγγιση στην Παρουσίαση Ερωτημάτων:**

Ένα βέλτιστο ερώτημα θα πρέπει να μεγιστοποιεί το κέρδος πληροφορίας:
```
Query* = argmax I(Y; H_response | Query, X)
```

Όπου I(·;·) είναι η αμοιβαία πληροφορία μεταξύ της πραγματικής ετικέτας Y και της ανθρώπινης απόκρισης H_response.

**Μοντελοποίηση Γνωσιακού Φορτίου:**
Η ποιότητα της ανθρώπινης απόκρισης υποβαθμίζεται με την πολυπλοκότητα:
```
P(correct_response) = f(Query_complexity, Context, User_expertise)
```

## 4. Μετρικές Αξιολόγησης για Συστήματα HITL

### 4.1 Μετρικές Επιπέδου Συστήματος

**Απόδοση Ομάδας Ανθρώπου-ΤΝ:**
- **Συμπληρωματικότητα (Complementarity)**: Απόδοση Ανθρώπου-ΤΝ > max(Απόδοση Ανθρώπου, Απόδοση ΤΝ)
- **Συνέργεια (Synergy)**: Απόδοση Ανθρώπου-ΤΝ > Απόδοση Ανθρώπου + Απόδοση ΤΝ - baseline

**Μετρικές Αποδοτικότητας:**
- **Αποδοτικότητα Ερωτημάτων**: Κέρδος απόδοσης ανά ανθρώπινο ερώτημα.
- **Λόγος Κόστους-Οφέλους**: (Βελτίωση_Απόδοσης × Αξία) / Ανθρώπινο_Κόστος.
- **Ρυθμός Μάθησης**: Βελτίωση στην απόδοση της ΤΝ με την πάροδο του χρόνου μέσω της ανθρώπινης ανατροφοδότησης.

### 4.2 Μετρικές Ποιότητας Αβεβαιότητας

**Βαθμονόμηση Αβεβαιότητας:**
- **Αξιοπιστία (Reliability)**: Αντιστοιχία μεταξύ εμπιστοσύνης και ακρίβειας.
- **Οξύτητα (Sharpness)**: Συγκέντρωση των προβλεπτικών κατανομών.
- **Διακριτική Ικανότητα (Resolution)**: Ικανότητα διάκρισης μεταξύ σωστών/λανθασμένων προβλέψεων.

**Ποιότητα Παρέμβασης:**
- **Ακρίβεια (Precision)**: Κλάσμα των παρεμβάσεων που ήταν απαραίτητες.
- **Ανάκληση (Recall)**: Κλάσμα των απαραίτητων παρεμβάσεων που όντως πραγματοποιήθηκαν.
- **F1-Score**: Αρμονικός μέσος της ακρίβειας και της ανάκλησης για τις παρεμβάσεις.

### 4.3 Μετρικές Ανθρώπινων Παραγόντων

**Εμπειρία Χρήστη:**
- **Γνωσιακό Φορτίο**: Μετριέται μέσω του χρόνου ολοκλήρωσης εργασιών και των ποσοστών σφάλματος.
- **Βαθμονόμηση Εμπιστοσύνης**: Κατάλληλη εξάρτηση από τις συστάσεις του συστήματος.
- **Επίδραση Μάθησης**: Βελτίωση της ανθρώπινης απόδοσης με την πάροδο του χρόνου.

**Κατανομή Φόρτου Εργασίας:**
- **Αποδοτικότητα Κατανομής Εργασιών**: Βέλτιστη κατανομή μεταξύ ανθρώπου και ΤΝ.
- **Κόστος Εναλλαγής Πλαισίου (Context Switching Cost)**: Επιβάρυνση από τις ανθρώπινες παρεμβάσεις.
- **Παρατεταμένη Προσοχή**: Υποβάθμιση της ανθρώπινης απόδοσης με την πάροδο του χρόνου.

### 4.4 Μετρικές Υιοθέτησης Χαρακτηριστικών (Νέο)

**Κρίσιμες για την ανίχνευση του «Ερωτώντας με Κακό Τρόπο»:**
- **Ποσοστό Απενεργοποίησης Χαρακτηριστικού**: Ποσοστό χρηστών που απενεργοποιούν τα «έξυπνα» χαρακτηριστικά.
- **Ποσοστό Χειροκίνητης Παράκαμψης**: Πόσο συχνά οι χρήστες αντιτίθενται στις αποφάσεις του συστήματος.
- **Ποσοστό Σιωπηλού Σφάλματος**: Σφάλματα που περνούν απαρατήρητα από τους χρήστες (δύσκολο να μετρηθεί, αλλά κρίσιμο).

## 5. Μελέτη Περίπτωσης: Ανίχνευση Δέσμευσης του Netflix (Περίπτωση Επιτυχίας)

### 5.1 Διατύπωση Προβλήματος

**Μεταβλητές Κατάστασης:**
- t: Τρέχων χρόνος θέασης.
- u: Ιστορικό αλληλεπίδρασης χρήστη.
- c: Τύπος περιεχομένου και μεταδεδομένα.
- h: Ιστορικά μοτίβα θέασης.

**Πρόβλημα Απόφασης:**
Προσδιορισμός του βέλτιστου χρόνου παρέμβασης T* για την ελαχιστοποίηση του:
```
Κόστος = α·P(user_absent|t)·Streaming_cost(t) + β·P(user_present|t)·Interruption_cost
```

### 5.2 Προσέγγιση Υλοποίησης

**Μηχανική Χαρακτηριστικών (Feature Engineering):**
```python
features = [
    'time_since_last_interaction',
    'hour_of_day', 
    'day_of_week',
    'content_type',
    'episode_number',
    'user_binge_history',
    'device_type',
    'viewing_session_length'
]
```

**Αρχιτεκτονική Μοντέλου:**
- **Αναδρομικό Νευρωνικό Δίκτυο (RNN)**: Αποτυπώνει χρονικές εξαρτήσεις.
- **Μηχανισμός Προσοχής (Attention Mechanism)**: Σταθμίζει την πρόσφατη έναντι της ιστορικής συμπεριφοράς.
- **Εκτίμηση Αβεβαιότητας**: Bayesian επίπεδα για διαστήματα εμπιστοσύνης.

**Βελτιστοποίηση Κατωφλίου:**
Πλαίσιο δοκιμών A/B για τη βελτιστοποίηση του χρονισμού της παρέμβασης:
```
Ομάδα Α: κατώφλι = t₁
Ομάδα Β: κατώφλι = t₂
Μέτρηση: Ικανοποίηση χρήστη, κόστος εύρους ζώνης, ποσοστά επαναδέσμευσης
```

### 5.3 Αποτελέσματα και Ανάλυση

**Μετρικές Απόδοσης:**
- **Μείωση Εύρους Ζώνης**: 15-20% μείωση στο περιττό streaming.
- **Ικανοποίηση Χρήστη**: Καμία σημαντική μείωση στις βαθμολογίες δέσμευσης.
- **Ποσοστό Ψευδώς Θετικών**: <5% διακοπές ενεργών χρηστών.
- **Ποσοστό Ψευδώς Αρνητικών**: <8% συνέχιση streaming σε απόντες χρήστες.

### 5.4 Γιατί το Netflix Επιτυγχάνει

**Βασικές Σχεδιαστικές Αποφάσεις:**
1. **Ρητή Επικοινωνία**: Κάνει παύση και ρωτά αντί να ενεργεί σιωπηλά.
2. **Αποτυχία Χαμηλού Ρίσκου**: Μια λάθος διακοπή κοστίζει δευτερόλεπτα, όχι ώρες.
3. **Εύκολη Επαναφορά**: Το πάτημα ενός κουμπιού συνεχίζει τη θέαση.
4. **Διαφανής Λογική**: Οι χρήστες κατανοούν τον κανόνα των 3 επεισοδίων/90 λεπτών.

## 6. Μελέτη Περίπτωσης: Home/Away Assist του Θερμοστάτη Nest (Περίπτωση Αποτυχίας)

### 6.1 Διατύπωση Προβλήματος

**Μεταβλητές Κατάστασης:**
- m: Ενδείξεις αισθητήρων κίνησης (δυαδικές ανά δωμάτιο).
- g: Τοποθεσία GPS των εγγεγραμμένων τηλεφώνων.
- t: Χρόνος από την τελευταία ανιχνευθείσα δραστηριότητα.
- s: Ιστορικό πρόγραμμα πληρότητας.
- w: Εξωτερικές καιρικές συνθήκες.

**Πρόβλημα Απόφασης:**
Προσδιορισμός της κατάστασης πληρότητας O ∈ {Home, Away} για τη βελτιστοποίηση του:
```
Κόστος = α·P(Away|O=Home)·Comfort_cost + β·P(Home|O=Away)·Energy_cost
```

### 6.2 Η Τεχνική Πρόκληση

**Περιορισμοί Αισθητήρων:**
```python
# Οι αισθητήρες κίνησης έχουν σημαντικά τυφλά σημεία
motion_coverage = {
    'living_room': 0.7,  # 70% του δωματίου ορατό
    'hallway': 0.9,
    'bedroom': 0.0,      # Κανένας αισθητήρας
    'home_office': 0.0   # Κανένας αισθητήρας
}

# Το GPS του τηλεφώνου είναι ανακριβές και ενεργοβόρο
gps_accuracy = {
    'outdoor': ±10m,
    'indoor': ±50m,      # Συχνά άχρηστο
    'basement': None     # Κανένα σήμα
}
```

**Πηγές Αβεβαιότητας (κυρίως γνωσιακές):**
1. **Μερική Παρατηρησιμότητα**: Δεν μπορεί να «δει» ολόκληρο το σπίτι.
2. **Θόρυβος Αισθητήρων**: Ανίχνευση κίνησης ≠ παρουσία ανθρώπου (κατοικίδια, κλιματισμός).
3. **Αλλαγή Συμπεριφοράς**: Η εργασία από το σπίτι ανατρέπει τα ιστορικά μοτίβα.
4. **Ασάφεια Πολλαπλών Ενοίκων**: Ένα τηλέφωνο που φεύγει ≠ άδειο σπίτι.

### 6.3 Πού Αποτυγχάνει ο Σχεδιασμός

**Το Κεντρικό Πρόβλημα: Δράση βάσει Αβεβαιότητας χωρίς Επικοινωνία**

```python
# ΠΡΑΓΜΑΤΙΚΗ ΛΟΓΙΚΗ NEST (απλοποιημένη)
def nest_decide(confidence_away, threshold=0.6):
    if confidence_away > threshold:
        switch_to_away()  # Σιωπηλή ενέργεια!
    else:
        stay_home()
    # Καμία ειδοποίηση, καμία επιβεβαίωση, καμία εισαγωγή από τον χρήστη
```

**Σύγκριση με το Netflix:**
```python
# ΛΟΓΙΚΗ NETFLIX
def netflix_decide(confidence_absent, threshold=0.7):
    if confidence_absent > threshold:
        pause_and_ask()  # Ρητή ερώτηση!
    else:
        continue_playing()
```

**Βασική Διαφορά:** Το Netflix αντιμετωπίζει την υπέρβαση του κατωφλίου ως έναυσμα για *ερώτηση*. Ο Nest την αντιμετωπίζει ως έναυσμα για *δράση*.

### 6.4 Ποσοτικοποίηση της Αποτυχίας

**Υποθετικές Μετρικές (βάσει ανάλυσης φόρουμ χρηστών):**
- **Ποσοστό Απενεργοποίησης Χαρακτηριστικού**: ~30% των χρηστών απενεργοποιούν το Home/Away Assist.
- **Ποσοστό Ψευδώς Θετικών (Away ενώ είναι Home)**: ~15-25% των αλλαγών λειτουργίας.
- **Χρόνος Επαναφοράς**: Κατά μέσο όρο 2+ ώρες προτού ο χρήστης το παρατηρήσει και το διορθώσει.
- **Ικανοποίηση Χρήστη**: Χαμηλή (εκτεταμένα παράπονα σε φόρουμ).

**Πρόβλημα Ασυμμετρίας Κόστους:**
```python
# Έμμεσο μοντέλο κόστους του Nest
C_comfort = minor  # Το σύστημα το θεωρεί χαμηλό κόστος
C_energy = major   # Το σύστημα δίνει προτεραιότητα στην εξοικονόμηση ενέργειας

# Πραγματικό μοντέλο κόστους του χρήστη
C_comfort = major  # Οι χρήστες μισούν να κρυώνουν/ζεσταίνονται στο σπίτι
C_energy = minor   # Η μικρή εξοικονόμηση δεν αξίζει την δυσφορία
```

### 6.5 Βελτιωμένος Σχεδιασμός (Τεχνικές Προδιαγραφές)

**Κλιμακωτή Απόκριση βάσει Εμπιστοσύνης:**

```python
def improved_nest_decide(confidence_away, stakes='normal'):
    # Προσαρμογή κατωφλίων βάσει του διακυβεύματος
    if stakes == 'extreme_weather':
        thresholds = {'act': 0.98, 'notify': 0.90, 'ask': 0.70}
    else:
        thresholds = {'act': 0.95, 'notify': 0.80, 'ask': 0.60}
    
    if confidence_away > thresholds['act']:
        switch_to_away()
        send_notification("Μετάβαση σε λειτουργία Απουσίας")
    elif confidence_away > thresholds['notify']:
        switch_to_away()
        send_notification("Μετάβαση σε λειτουργία Απουσίας. Πατήστε αν είστε στο σπίτι.")
    elif confidence_away > thresholds['ask']:
        send_notification("Δεν ανιχνεύθηκε δραστηριότητα. Μετάβαση σε λειτουργία Απουσίας;",
                         options=["Ναι", "Όχι", "Ρώτα αργότερα"])
        # Αναμονή για απάντηση πριν την ενέργεια
    else:
        stay_home()  # Ανεπαρκής εμπιστοσύνη για δράση ή ερώτηση
```

**Ενσωμάτωση Ανατροφοδότησης:**
```python
class ImprovedNestLearner:
    def __init__(self):
        self.false_positive_count = 0
        self.false_negative_count = 0
        self.user_patterns = {}
    
    def update_from_correction(self, predicted, actual, context):
        if predicted == 'Away' and actual == 'Home':
            self.false_positive_count += 1
            # Αύξηση κατωφλίου για αυτή την ώρα/ημέρα
            self.user_patterns[context['day_hour']] = 'likely_home'
        
        # Προσαρμογή κατωφλίων βάσει του ιστορικού σφαλμάτων
        if self.false_positive_count > 5:
            self.ask_threshold *= 1.1  # Γίνε πιο συντηρητικός
```

### 6.6 Μαθήματα για τον Σχεδιασμό Συστημάτων HITL

| Αρχή | Υλοποίηση Netflix | Υλοποίηση Nest | Σύσταση |
|-----------|----------------------|---------------------|----------------|
| **Αβεβαιότητα → Δράση** | Παύση και ερώτηση | Σιωπηλή δράση | Ερώτηση πριν τη δράση |
| **Επαναφορά Σφάλματος** | Άμεση (ένα κουμπί) | Καθυστερημένη (ώρες) | Καταστήστε την επαναφορά απλή |
| **Διαφάνεια** | Σαφείς κανόνες (3 επ./90 λεπ.) | Αδιαφανής αλγόριθμος | Εξήγηση του σκεπτικού |
| **Ορατή Μάθηση** | Έμμεση αλλά γρήγορη | Κρυφή και αργή | Δείξτε τι έμαθε το σύστημα |
| **Αντιστοίχιση Ρίσκου** | Χαμηλό ρίσκο, απλή ερώτηση | Υψηλό ρίσκο, καμία ερώτηση | Ταίριασμα επείγοντος με αντίκτυπο |

## 7. Επεκτάσεις και Μελλοντικές Κατευθύνσεις

### 7.1 Μετα-Μάθηση (Meta-Learning) για Αποφάσεις Παρέμβασης

**Μαθαίνοντας να Ζητά Βοήθεια:**
Εκπαίδευση μετα-μοντέλων που μαθαίνουν πότε η ανθρώπινη παρέμβαση είναι πιο πολύτιμη:
```
Meta-model: p(intervention_value | task_features, model_state, human_expertise)
```

### 7.2 Πολυτροπική Συγχώνευση Αβεβαιότητας (Multi-Modal Uncertainty Fusion)

Συνδυασμός εκτιμήσεων αβεβαιότητας από διαφορετικές πηγές:
```
σ²_total = Σ w_i · σ²_i + λ · Disagreement(modalities)
```

### 7.3 Εξατομικευμένα Κατώφλια Αβεβαιότητας

Βαθμονόμηση ανάλογα με τον χρήστη βάσει:
- **Ανοχής στο Ρίσκο**: Ατομική άνεση με την αυτονομία του συστήματος.
- **Επιπέδου Εμπειρογνωμοσύνης**: Η γνώση του τομέα επηρεάζει τη βέλτιστη συχνότητα παρέμβασης.
- **Πλαισίου (Context)**: Η κρισιμότητα της εργασίας επηρεάζει τις ρυθμίσεις των κατωφλίων.

### 7.4 Σχεδιασμός HITL με Επίγνωση Επικοινωνίας

Νέα ερευνητική κατεύθυνση εμπνευσμένη από τη σύγκριση Netflix vs. Nest:
```
Optimize: min E[Cost] + λ · Communication_Cost
Subject to: User_Trust ≥ threshold
           Feature_Disable_Rate ≤ ε
```

Όπου Communication_Cost περιλαμβάνει:
- Συχνότητα ειδοποιήσεων.
- Πολυπλοκότητα ερωτημάτων.
- Επιβάρυνση απόκρισης.

### Θεωρητικές Ερωτήσεις
1. Πώς αντιστοιχίζονται οι τύποι αβεβαιότητας (τυχαία έναντι γνωσιακής) σε διαφορετικές στρατηγικές παρέμβασης;
2. Ποια είναι τα θεωρητικά όρια της ποσοτικοποίησης αβεβαιότητας σε μοντέλα βαθιάς μάθησης;
3. Πώς μπορούμε να αποδείξουμε τυπικά τη βελτιστοποίηση των κατωφλίων ανθρώπινης παρέμβασης;
4. **Νέο:** Πώς θα πρέπει να αλλάξει το πλαίσιο απόφασης όταν το κόστος της «σιωπηλής δράσης» υπερβαίνει το κόστος της «ερώτησης»;

### Πρακτικές Ερωτήσεις
1. Πώς σχεδιάζετε δοκιμές A/B για συστήματα HITL όπου η ανθρώπινη μάθηση επηρεάζει τα αποτελέσματα;
2. Ποιες είναι οι ηθικές επιπτώσεις των συστημάτων λήψης αποφάσεων βάσει αβεβαιότητας σε τομείς υψηλού ρίσκου;
3. Πώς λαμβάνετε υπόψη τις ανθρώπινες γνωσιακές προκαταλήψεις σε διεπαφές που βασίζονται στην αβεβαιότητα;
4. **Νέο:** Ποιες μετρικές θα ανίχνευαν μια αποτυχία τύπου «Nest» (ερώτηση με κακό τρόπο) προτού οι χρήστες απενεργοποιήσουν το χαρακτηριστικό;

### Ερευνητικές Κατευθύνσεις
1. Μπορούμε να αναπτύξουμε ενεργό μάθηση με επίγνωση αβεβαιότητας που να λαμβάνει υπόψη τα επίπεδα ανθρώπινης εμπειρογνωμοσύνης;
2. Πώς βελτιστοποιούμε την απόδοση της ομάδας ανθρώπου-ΤΝ μακροπρόθεσμα έναντι των βραχυπρόθεσμων μετρικών;
3. Ποιες νέες μέθοδοι ποσοτικοποίησης αβεβαιότητας απαιτούνται για πολυτροπικά συστήματα ΤΝ;
4. **Νέο:** Πώς μπορούν τα συστήματα να μάθουν να προβλέπουν πότε οι χρήστες θα απενεργοποιήσουν τα «έξυπνα» χαρακτηριστικά και να βελτιώνονται προληπτικά;
