## 1. Πλαίσια Ανάλυσης Σφαλμάτων

Η τυπική αξιολόγηση ML — ακρίβεια, πιστότητα, ανάκληση, F1, AUC — μετρά τη συνολική απόδοση του μοντέλου. Στα συστήματα HITL, οι συνολικές μετρικές είναι απαραίτητες αλλά όχι επαρκείς. Ένα μοντέλο με ακρίβεια 95% μπορεί να έχει ακόμα συστηματικές μορφές αποτυχίας που είναι αόρατες στις συνολικές στατιστικές.

### 1.1 Ανάλυση Πίνακα Σύγχυσης

Το βασικό δομικό στοιχείο της ανάλυσης σφαλμάτων είναι ο πίνακας σύγχυσης. Για έναν δυαδικό ταξινομητή:

|  | Προβλεπόμενο Θετικό | Προβλεπόμενο Αρνητικό |
|---|---|---|
| **Πραγματικό Θετικό** | Αληθώς Θετικό (TP) | Ψευδώς Αρνητικό (FN) |
| **Πραγματικό Αρνητικό** | Ψευδώς Θετικό (FP) | Αληθώς Αρνητικό (TN) |

Στα συστήματα HITL, η ασυμμετρία μεταξύ του κόστους FP και FN είναι κρίσιμη για τον σχεδιασμό των ορίων. Για ένα σύστημα ιατρικής διάγνωσης, ένα Ψευδώς Αρνητικό (διαφυγούσα ασθένεια) είναι συνήθως πολύ πιο δαπανηρό από ένα Ψευδώς Θετικό (περιττή επανεξέταση). Αυτή η ασυμμετρία θα πρέπει να καθορίζει άμεσα το πού τίθεται το όριο.

Το βέλτιστο όριο υπό ασύμμετρα κόστη:

$$\theta^* = \frac{C_{FP}}{C_{FP} + C_{FN}}$$

όπου $C_{FP}$ είναι το κόστος ενός ψευδώς θετικού και $C_{FN}$ είναι το κόστος ενός ψευδώς αρνητικού.

Για ένα σύστημα προσυμπτωματικού ελέγχου καρκίνου με $C_{FP} = \$200$ (περιττή βιοψία) και $C_{FN} = \$50.000$ (διαφυγούσα διάγνωση):

$$\theta^* = \frac{200}{200 + 50.000} \approx 0,004$$

Αυτό σημαίνει ότι το σύστημα θα πρέπει να επισημαίνει για ανθρώπινη αξιολόγηση κάθε φορά που η εμπιστοσύνη του μοντέλου στην αρνητική πρόβλεψη υπερβαίνει το 99,6% — δηλαδή, θα πρέπει να είναι εξαιρετικά συντηρητικό σχετικά με τις αυτοματοποιημένες αρνητικές αποφάσεις.

### 1.2 Ανάλυση Βαθμονόμησης

Ένα καλά βαθμονομημένο μοντέλο είναι αυτό του οποίου τα σκορ εμπιστοσύνης αντιστοιχούν στην εμπειρική ακρίβεια: ένα μοντέλο που λέει "εμπιστοσύνη 90%" θα πρέπει να είναι σωστό περίπου στο 90% των περιπτώσεων.

Η βαθμονόμηση αξιολογείται με **διαγράμματα αξιοπιστίας**: απεικονίστε την προβλεπόμενη εμπιστοσύνη στον άξονα x έναντι της πραγματικής ακρίβειας για αυτό το επίπεδο εμπιστοσύνης στον άξονα y. Ένα τέλεια βαθμονομημένο μοντέλο παράγει μια διαγώνια γραμμή. Η καμπύλη ενός υπερβολικά σίγουρου μοντέλου πέφτει κάτω από τη διαγώνιο (λέει 80% εμπιστοσύνη αλλά είναι σωστό μόνο στο 60% των περιπτώσεων). Η καμπύλη ενός λιγότερο σίγουρου μοντέλου πέφτει πάνω από αυτήν.

Η τυπική βαθμωτή μετρική είναι το **Αναμενόμενο Σφάλμα Βαθμονόμησης (ECE)**:

$$ECE = \sum_{b=1}^{B} \frac{|B_b|}{n} \left| \text{acc}(B_b) - \text{conf}(B_b) \right|$$

όπου $B_b$ είναι ένα δοχείο (bucket) προβλέψεων ομαδοποιημένων κατά εμπιστοσύνη, $\text{acc}(B_b)$ είναι η εμπειρική ακρίβεια σε αυτό το δοχείο και $\text{conf}(B_b)$ είναι η μέση εμπιστοσύνη.

**Γιατί αυτό έχει σημασία για το HITL**: Εάν ένα μοντέλο είναι συστηματικά υπερβολικά σίγουρο, οι εκτιμήσεις αβεβαιότητάς του δεν μπορούν να είναι έμπιστες για την ενεργοποίηση της ανθρώπινης αξιολόγησης στις σωστές στιγμές. Η βαθμονόμηση είναι προϋπόθεση για την ουσιαστική ανίχνευση αβεβαιότητας.

### 1.3 Ανάλυση Χρονικής Σταθερότητας

Τα μοντέλα που εκπαιδεύονται τη χρονική στιγμή $t$ θα αντιμετωπίσουν μετατόπιση κατανομής καθώς ο κόσμος αλλάζει. Στα συστήματα HITL, μια χρήσιμη διάγνωση είναι η σύγκριση της απόδοσης του μοντέλου με την πάροδο του χρόνου:

```python
def temporal_stability_analysis(predictions_log, window_days=30):
    """
    Διαχωρισμός του αρχείου προβλέψεων σε χρονικά παράθυρα.
    Σύγκριση της ακρίβειας μεταξύ των παραθύρων για την ανίχνευση διολίσθησης.
    """
    df = pd.DataFrame(predictions_log)
    df['date'] = pd.to_datetime(df['timestamp'])
    df['window'] = df['date'].dt.to_period(f'{window_days}D')
    
    results = df.groupby('window').agg({
        'correct': ['mean', 'count'],
        'confidence': 'mean'
    })
    
    # Επισήμανση παραθύρων όπου η ακρίβεια πέφτει περισσότερο από 2 τυπικές αποκλίσεις κάτω από τη βάση
    baseline_acc = results['correct']['mean'][:3].mean()
    baseline_std = results['correct']['mean'][:3].std()
    
    results['drift_flag'] = (
        results['correct']['mean'] < baseline_acc - 2 * baseline_std
    )
    
    return results
```

## 2. Αξιολόγηση βάσει Τμημάτων (Slice-Based Evaluation)

Η πιο ισχυρή τεχνική για την ανίχνευση συστηματικών αποτυχιών είναι η αξιολόγηση βάσει τμημάτων: αντί για τον υπολογισμό μιας συνολικής μετρικής, υπολογίστε μετρικές ξεχωριστά για σημαντικούς υποπληθυσμούς.

### 2.1 Ορισμός Τμημάτων (Slices)

Τα τμήματα θα πρέπει να ορίζονται με βάση:
- **Δημογραφικά στοιχεία**: Ηλικία, φύλο, φυλή/εθνότητα, γεωγραφική περιοχή, γλώσσα
- **Χαρακτηριστικά ειδικά για τον τομέα**: Στην ιατρική ΤΝ, σοβαρότητα νόσου, τύπος εξοπλισμού απεικόνισης, ίδρυμα
- **Χρονικά χαρακτηριστικά**: Ώρα της ημέρας, ημέρα της εβδομάδας, εποχή, έτος
- **Χαρακτηριστικά εισόδου**: Μήκος κειμένου, ποιότητα εικόνας, ποσό συναλλαγής

Στην ανίχνευση απάτης, τα σχετικά τμήματα μπορεί να περιλαμβάνουν:
- Εύρος ποσού συναλλαγής ($0–$100, $100–$1000, $1000+)
- Κατηγορία εμπόρου (λιανική, ταξίδια, ψηφιακά αγαθά, διασυνοριακά)
- Διάρκεια κατοχής λογαριασμού (νέος πελάτης, < 1 έτος, 1–5 έτη, > 5 έτη)
- Γεωγραφική προέλευση της συναλλαγής

### 2.2 Ανάλυση Απόδοσης Τμημάτων

```python
def slice_performance_analysis(df, target_col, pred_col, slice_cols):
    """
    Υπολογισμός μετρικών ακρίβειας για κάθε συνδυασμό τιμών τμημάτων.
    Επισήμανση τμημάτων όπου η απόδοση πέφτει κάτω από το όριο.
    """
    results = []
    
    # Απόδοση βάσης
    baseline = compute_metrics(df[target_col], df[pred_col])
    
    for col in slice_cols:
        for val in df[col].unique():
            slice_df = df[df[col] == val]
            if len(slice_df) < 50:  # Ελάχιστο μέγεθος δείγματος
                continue
            
            metrics = compute_metrics(
                slice_df[target_col], 
                slice_df[pred_col]
            )
            
            results.append({
                'slice_col': col,
                'slice_val': val,
                'n': len(slice_df),
                'accuracy': metrics['accuracy'],
                'gap_from_baseline': metrics['accuracy'] - baseline['accuracy'],
                'precision': metrics['precision'],
                'recall': metrics['recall']
            })
    
    return pd.DataFrame(results).sort_values('gap_from_baseline')
```

### 2.3 Διατομεακή Τμηματοποίηση (Intersectional Slicing)

Η μονοδιάστατη τμηματοποίηση παραλείπει τις διατομεακές αποτυχίες. Ένα μοντέλο μπορεί να αποδίδει καλά σε κάθε μεμονωμένο χαρακτηριστικό, αλλά να αποτυγχάνει σε συνδυασμούς: καλά στις "γυναίκες" και καλά στο "υψηλό εισόδημα", αλλά άσχημα στις "γυναίκες με υψηλό εισόδημα". Ο χώρος των διατομεακών τμημάτων αυξάνεται εκθετικά, απαιτώντας είτε επιλογή καθοδηγούμενη από τον τομέα είτε αυτοματοποιημένες μεθόδους.

Οι μέθοδοι **εύρεσης τμημάτων (slice finder)** (Chung et al., 2019; Eyuboglu et al., 2022) χρησιμοποιούν προσεγγίσεις βασισμένες σε συσταδοποίηση (clustering) και δέντρα απόφασης για τον αυτόματο προσδιορισμό υποπληθυσμών όπου η απόδοση του μοντέλου είναι χαμηλή.

## 3. Συμπεριφορικός Έλεγχος: Η Μεθοδολογία CheckList

Η μεθοδολογία CheckList (Ribeiro et al., 2020, Καλύτερη Εργασία ACL) προσαρμόζει τις αρχές του ελέγχου λογισμικού στα μοντέλα NLP. Αντί να ρωτά "ποια είναι η συνολική ακρίβεια αυτού του μοντέλου;", ρωτά "ικανοποιεί αυτό το μοντέλο συγκεκριμένες συμπεριφορικές αναλλοίωτες (invariances);".

### 3.1 Τρεις Τύποι Ελέγχου

**Έλεγχοι Ελάχιστης Λειτουργικότητας (Minimum Functionality Tests - MFT)**: Χειρίζεται το μοντέλο σωστά τις βασικές, προφανείς περιπτώσεις;

```python
# Για έναν ταξινομητή συναισθήματος
mft_tests = [
    ("Αυτό το προϊόν είναι εξαιρετικό!", "θετικό"),
    ("Το μισώ αυτό τόσο πολύ.", "αρνητικό"),
    ("Το αντικείμενο έφτασε στην ώρα του.", "ουδέτερο"),
]
```

**Έλεγχοι Αναλλοίωτης (Invariance Tests - INV)**: Παραμένει σταθερή η έξοδος του μοντέλου όταν οι είσοδοι αλλάζουν με τρόπους που δεν θα έπρεπε να επηρεάζουν την έξοδο;

```python
# Το συναίσθημα δεν θα πρέπει να αλλάζει όταν εναλλάσσονται τα ονόματα
inv_tests = [
    ("Ο Γιάννης λάτρεψε το προϊόν.", "Η Μαρία λάτρεψε το προϊόν."),  # αναμένεται η ίδια ετικέτα
    ("Το προϊόν απογοήτευσε τη Σάρα.", "Το προϊόν απογοήτευσε τον Μπάμπη."),  # αναμένεται η ίδια ετικέτα
]
```

**Κατευθυντικοί Έλεγχοι (Directional Tests - DIR)**: Αλλάζει η έξοδος του μοντέλου κατάλληλα όταν οι είσοδοι αλλάζουν με τρόπους που θα έπρεπε να επηρεάζουν την έξοδο;

```python
# Το συναίσθημα θα πρέπει να γίνει πιο αρνητικό με την προσθήκη άρνησης
dir_tests = [
    ("Το προϊόν είναι καλό.", "Το προϊόν δεν είναι καλό."),  # θα πρέπει να μειωθεί
    ("Το συνιστώ αυτό.", "Δεν το συνιστώ αυτό."),  # θα πρέπει να μειωθεί
]
```

### 3.2 Εφαρμογή του CheckList στην Ποιότητα Σχολιασμού HITL

Το CheckList μπορεί να προσαρμοστεί για τον έλεγχο της ποιότητας του σχολιασμού αντί για την ποιότητα του μοντέλου. Για τον συντονισμό περιεχομένου:

```python
def annotation_behavioral_tests(annotation_sample):
    """
    Έλεγχος της συνέπειας του σχολιασμού σε συμπεριφορικές διαταραχές.
    """
    # Αναλλοίωτη: Ίδιο περιεχόμενο, αναφορά σε διαφορετικό προστατευόμενο χαρακτηριστικό
    inv_pairs = [
        ("Η ομιλία από την [Ομάδα Α] ήταν προσβλητική", 
         "Η ομιλία από την [Ομάδα Β] ήταν προσβλητική"),
    ]
    
    consistency_scores = []
    for text_a, text_b in inv_pairs:
        labels_a = annotation_sample.get_labels(text_a)
        labels_b = annotation_sample.get_labels(text_b)
        consistency_scores.append(
            cohen_kappa_score(labels_a, labels_b)
        )
    
    return {
        'mean_consistency': np.mean(consistency_scores),
        'min_consistency': np.min(consistency_scores),
        'flagged_pairs': [
            pair for pair, score 
            in zip(inv_pairs, consistency_scores) 
            if score < 0.6  # Κάτω από το αποδεκτό όριο
        ]
    }
```

### 3.3 CheckList για τη Συμπεριφορά των Αξιολογητών

Το ίδιο πλαίσιο ισχύει για τον έλεγχο της συνέπειας των ανθρώπινων αξιολογητών σε συστήματα HITL:

- **Αντίστοιχο MFT**: Χειρίζονται οι αξιολογητές σωστά τις προφανείς περιπτώσεις (σαφείς παραβιάσεις, σαφείς μη παραβιάσεις);
- **Αντίστοιχο INV**: Εφαρμόζουν οι αξιολογητές το ίδιο κριτήριο σε ισοδύναμο περιεχόμενο από διαφορετικές δημογραφικές ομάδες, πολιτικές οπτικές ή στυλ γραφής;
- **Αντίστοιχο DIR**: Είναι οι αξιολογητές κατάλληλα πιο πιθανό να επισημάνουν περιεχόμενο καθώς αυξάνεται η σοβαρότητα της παραβίασης;

## 4. Αποσφαλμάτωση Δεδομένων με Συναρτήσεις Επιρροής

Όταν εντοπίζετε μια συστηματική αποτυχία του μοντέλου, πρέπει να την ιχνηλατήσετε στην πηγή της στα δεδομένα εκπαίδευσης. Οι συναρτήσεις επιρροής παρέχουν μια τεκμηριωμένη μέθοδο για να ρωτήσετε: ποια παραδείγματα εκπαίδευσης επηρέασαν περισσότερο την πρόβλεψη του μοντέλου σε μια δεδομένη περίπτωση δοκιμής;

### 4.1 Η Έννοια της Συνάρτησης Επιρροής

Η συνάρτηση επιρροής (Koh & Liang, 2017) εκτιμά τη μεταβολή στις παραμέτρους του μοντέλου εάν ένα μεμονωμένο παράδειγμα εκπαίδευσης αφαιρεθεί ή αυξηθεί το βάρος του:

$$\mathcal{I}(z, z_{test}) = -\nabla_\theta \mathcal{L}(z_{test}, \hat{\theta})^T H_{\hat{\theta}}^{-1} \nabla_\theta \mathcal{L}(z, \hat{\theta})$$

όπου $z$ είναι ένα παράδειγμα εκπαίδευσης, $z_{test}$ είναι το παράδειγμα δοκιμής ενδιαφέροντος, $\hat{\theta}$ είναι οι εκπαιδευμένες παράμετροι, $H_{\hat{\theta}}$ είναι ο Εσσιανός (Hessian) πίνακας της απώλειας εκπαίδευσης και $\mathcal{L}$ είναι η συνάρτηση απώλειας.

Στην πράξη, αυτό σας λέει: εάν το παράδειγμα εκπαίδευσης $z$ αφαιρούνταν, πόσο θα άλλαζε η πρόβλεψη του μοντέλου στο $z_{test}$;

### 4.2 Χρήση Συναρτήσεων Επιρροής για την Ανίχνευση Μεροληψίας

```python
from sklearn.linear_model import LogisticRegression
import numpy as np

def compute_influence_scores(model, X_train, y_train, X_test_point):
    """
    Απλοποιημένος υπολογισμός συνάρτησης επιρροής για γραμμικά μοντέλα.
    Για βαθιά μοντέλα, χρησιμοποιήστε το TracIn ή προσεγγιστικές μεθόδους επιρροής.
    """
    # Υπολογισμός κλίσης στο σημείο δοκιμής
    pred = model.predict_proba(X_test_point.reshape(1, -1))
    residual = y_test_point - pred[0, 1]
    grad_test = residual * X_test_point
    
    # Υπολογισμός Εσσιανού (για λογιστική παλινδρόμηση, χρησιμοποιήστε πληροφορία Fisher)
    preds_train = model.predict_proba(X_train)
    weights = preds_train[:, 0] * preds_train[:, 1]
    H = (X_train.T * weights) @ X_train / len(X_train)
    
    # Υπολογισμός επιρροής για κάθε σημείο εκπαίδευσης
    H_inv_grad_test = np.linalg.solve(H, grad_test)
    
    influences = np.array([
        -np.dot(residual_i * X_train[i], H_inv_grad_test)
        for i, residual_i in enumerate(y_train - preds_train[:, 1])
    ])
    
    return influences

def find_most_influential_training_examples(
    model, X_train, y_train, 
    problematic_test_examples, 
    top_k=50
):
    """
    Για ένα σύνολο προβληματικών παραδειγμάτων δοκιμής,
    βρείτε τα παραδείγματα εκπαίδευσης που είναι περισσότερο υπεύθυνα.
    """
    aggregate_influence = np.zeros(len(X_train))
    
    for x_test in problematic_test_examples:
        influences = compute_influence_scores(
            model, X_train, y_train, x_test
        )
        aggregate_influence += np.abs(influences)
    
    top_indices = np.argsort(aggregate_influence)[-top_k:]
    return X_train[top_indices], y_train[top_indices], top_indices
```

### 4.3 Πρακτική Εφαρμογή: Ιχνηλάτηση Θορύβου Ετικετών

Μια πιο πρακτική εφαρμογή σε σενάρια αποτυχίας σχολιασμού είναι η **ανίχνευση θορύβου ετικετών**. Τα παραδείγματα εκπαίδευσης με υψηλή απώλεια (loss) υπό το εκπαιδευμένο μοντέλο είναι υποψήφια για εσφαλμένη επισήμανση:

```python
def detect_potential_mislabels(model, X_train, y_train, threshold_percentile=95):
    """
    Προσδιορισμός παραδειγμάτων εκπαίδευσης με ανώμαλα υψηλή απώλεια.
    Αυτά είναι υποψήφια για επανασήμανση.
    """
    preds = model.predict_proba(X_train)
    
    # Υπολογισμός απώλειας cross-entropy ανά παράδειγμα
    losses = -np.array([
        y_train[i] * np.log(preds[i, 1] + 1e-10) + 
        (1 - y_train[i]) * np.log(preds[i, 0] + 1e-10)
        for i in range(len(y_train))
    ])
    
    threshold = np.percentile(losses, threshold_percentile)
    suspicious_indices = np.where(losses > threshold)[0]
    
    return {
        'suspicious_indices': suspicious_indices,
        'suspicious_labels': y_train[suspicious_indices],
        'losses': losses[suspicious_indices],
        'recommended_for_review': X_train[suspicious_indices]
    }
```

## 5. Μετρικές Συμφωνίας Μεταξύ Σχολιαστών

Για τη διάγνωση της αποτυχίας σχολιασμού, η βασική μέτρηση είναι η συμφωνία μεταξύ των σχολιαστών. Αρκετές μετρικές είναι κατάλληλες ανάλογα με την εργασία:

**Cohen's Kappa** (δύο σχολιαστές, κατηγορικές ετικέτες):
$$\kappa = \frac{p_o - p_e}{1 - p_e}$$

όπου $p_o$ είναι η παρατηρούμενη συμφωνία και $p_e$ είναι η αναμενόμενη συμφωνία λόγω τύχης. Τιμές πάνω από 0,6 είναι γενικά αποδεκτές, πάνω από 0,8 είναι καλές, κάτω από 0,4 υποδηλώνουν σοβαρά προβλήματα σχολιασμού.

**Fleiss' Kappa** (περισσότεροι από δύο σχολιαστές):
$$\kappa_F = \frac{\bar{P} - \bar{P}_e}{1 - \bar{P}_e}$$

**Krippendorff's Alpha** (το πιο ευέλικτο, χειρίζεται ελλιπή δεδομένα και τακτικές κλίμακες):
$$\alpha = 1 - \frac{D_o}{D_e}$$

όπου $D_o$ είναι η παρατηρούμενη διαφωνία και $D_e$ είναι η αναμενόμενη διαφωνία.

### Ανίχνευση Διολίσθησης Σχολιαστών

```python
def detect_annotator_drift(annotation_log, window_size=500):
    """
    Έλεγχος αν η συμφωνία μεμονωμένων σχολιαστών με τη συναίνεση
    διολισθαίνει με την πάροδο του χρόνου (υποδηλώνοντας κόπωση ή
    μεταβαλλόμενα κριτήρια).
    """
    df = pd.DataFrame(annotation_log)
    df = df.sort_values('timestamp')
    
    drift_results = {}
    
    for annotator_id in df['annotator_id'].unique():
        annotator_df = df[df['annotator_id'] == annotator_id]
        
        # Υπολογισμός συμφωνίας με τη συναίνεση σε κυλιόμενα παράθυρα
        windows = []
        for i in range(0, len(annotator_df) - window_size, window_size // 2):
            window = annotator_df.iloc[i:i + window_size]
            kappa = cohen_kappa_score(
                window['label'], 
                window['consensus_label']
            )
            windows.append({
                'window_start': window['timestamp'].iloc[0],
                'kappa': kappa,
                'n': len(window)
            })
        
        drift_results[annotator_id] = pd.DataFrame(windows)
    
    return drift_results
```

## 6. Η Ροή Εργασίας Αποσφαλμάτωσης HITL

Τα τέσσερα παραπάνω εργαλεία είναι πιο ισχυρά όταν χρησιμοποιούνται στη σειρά:

1. **Ανάλυση βαθμονόμησης** → Είναι τα σκορ εμπιστοσύνης του μοντέλου αξιόπιστα; Εάν όχι, η ανίχνευση αβεβαιότητας είναι χαλασμένη.

2. **Αξιολόγηση βάσει τμημάτων** → Υπάρχουν υποπληθυσμοί όπου η απόδοση είναι συστηματικά χαμηλότερη; Εάν ναι, πρόκειται για αποτυχία μοντέλου ή αποτυχία σχολιασμού.

3. **Συμπεριφορικοί έλεγχοι CheckList** → Υπάρχουν συγκεκριμένες γλωσσικές ή δημογραφικές αναλλοίωτες που αποτυγχάνουν; Εάν ναι, είναι πιθανή η αποτυχία σχολιασμού.

4. **Συναρτήσεις επιρροής / Ανίχνευση θορύβου ετικετών** → Ποια συγκεκριμένα παραδείγματα εκπαίδευσης είναι υπεύθυνα; Δώστε προτεραιότητα για επανασήμανση.

5. **Ανάλυση συμφωνίας μεταξύ σχολιαστών** → Είναι η ίδια η διαδικασία επισήμανσης αξιόπιστη; Εάν όχι, ολόκληρος ο βρόχος ανάδρασης είναι ύποπτος.
