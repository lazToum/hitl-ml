## 4A.1 Η καμπύλη ROC και το AUC

### 4A.1.1 Ορισμός

Για έναν δυαδικό ταξινομητή με βαθμολογίες s(x) ∈ [0,1], η μεταβολή του κατωφλίου τ ∈ [0,1] ιχνηλατεί την καμπύλη ROC (Receiver Operating Characteristic):

```
TPR(τ) = P(s(x) ≥ τ | y = 1)   [Ποσοστό Αληθώς Θετικών / Ευαισθησία]
FPR(τ) = P(s(x) ≥ τ | y = 0)   [Ποσοστό Ψευδώς Θετικών]
```

Το Εμβαδόν κάτω από την Καμπύλη ROC (Area Under the ROC Curve - AUC) ισούται με την πιθανότητα μια τυχαία θετική περίπτωση να βαθμολογηθεί υψηλότερα από μια τυχαία αρνητική περίπτωση:

```
AUC = P(s(x+) > s(x-))
```

Το AUC μετρά την ποιότητα της κατάταξης, ανεξάρτητα από την επιλογή του κατωφλίου.

**Βασική παρατήρηση:** Το AUC σάς λέει πόσο διαχωρίσιμες είναι οι κλάσεις σας. Το κατώφλι σάς λέει πού να κάνετε την τομή δεδομένης της δομής του κόστους σας. Αυτά είναι ξεχωριστά ερωτήματα.

### 4A.1.2 Επιλογή Βέλτιστου Κατωφλίου

Το βέλτιστο κατώφλι τ* ελαχιστοποιεί το αναμενόμενο κόστος:

```
E[Cost(τ)] = C_FN · FNR(τ) · P(y=1) + C_FP · FPR(τ) · P(y=0)
```

Όπου:
- C_FN = κόστος ψευδώς αρνητικού (παράλειψη)
- C_FP = κόστος ψευδώς θετικού (ψευδής συναγερμός)
- P(y=1) = βασικό ποσοστό της θετικής κλάσης

Θέτοντας dE[Cost(τ)]/dτ = 0 προκύπτει το βέλτιστο κατώφλι:

```
τ* = C_FP · P(y=0) / (C_FP · P(y=0) + C_FN · P(y=1))
```

Για ίσα κόστη (C_FN = C_FP) και ισορροπημένες κλάσεις (P(y=0) = P(y=1) = 0,5): τ* = 0,5

Για C_FN >> C_FP (η παράλειψη θετικής περίπτωσης είναι πολύ δαπανηρή, π.χ. προσυμπτωματικός έλεγχος καρκίνου): τ* << 0,5

## 4A.2 Συμβιβασμός Ακρίβειας-Ανάκλησης (Precision-Recall)

### 4A.2.1 Πότε το AUC είναι παραπλανητικό

Το AUC κυριαρχείται από την απόδοση στην πλειοψηφούσα κλάση. Για προβλήματα με ανισορροπία (σπάνια θετική κλάση), η ανάλυση ακρίβειας-ανάκλησης είναι πιο κατατοπιστική.

```
Precision(τ) = TP / (TP + FP) = P(y=1 | s(x) ≥ τ)
Recall(τ)    = TP / (TP + FN) = P(s(x) ≥ τ | y=1)
```

Η **βαθμολογία F_β** εξισορροπεί την ακρίβεια (precision) και την ανάκληση (recall) με βάρος β:

```
F_β = (1 + β²) · Precision · Recall / (β² · Precision + Recall)
```

- β = 1: Ίσο βάρος σε ακρίβεια και ανάκληση (βαθμολογία F1)
- β = 2: Η ανάκληση είναι διπλάσιας σημασίας (χρήση όταν τα ψευδώς αρνητικά είναι πιο δαπανηρά)
- β = 0,5: Η ακρίβεια είναι διπλάσιας σημασίας (χρήση όταν τα ψευδώς θετικά είναι πιο δαπανηρά)

### 4A.2.2 Η ζώνη HITL ως γέφυρα Ακρίβειας-Ανάκλησης

Ένα σύστημα HITL δύο κατωφλίων (τ_low, τ_high) με ανθρώπινη αναθεώρηση για βαθμολογίες στο διάστημα (τ_low, τ_high):

```
Αυτοματοποιημένο Ποσοστό Αληθώς Θετικών = P(s(x) ≥ τ_high | y=1)
Αυτοματοποιημένο Ποσοστό Ψευδώς Θετικών = P(s(x) ≥ τ_high | y=0)
Ποσοστό Ανθρώπινης Αναθεώρησης = P(τ_low ≤ s(x) < τ_high)
```

Αυτό το σύστημα επιτυγχάνει πλήρη ακρίβεια (μόνο οι αναθεωρήσεις πάνω από τ_high είναι αυτοματοποιημένα θετικές) με το κόστος της αποστολής της αβέβαιης ζώνης σε ανθρώπους. Η βελτιστοποίηση του εύρους της ζώνης εξισορροπεί τον φόρτο εργασίας του αναθεωρητή έναντι των συμβιβασμών ακρίβειας/ανάκλησης.

## 4A.3 Μάθηση με Επίγνωση Κόστους (Cost-Sensitive Learning)

### 4A.3.1 Στάθμιση Κλάσεων

Τα περισσότερα πλαίσια Μηχανικής Μάθησης (ML) επιτρέπουν την εκπαίδευση με επίγνωση κόστους μέσω των βαρών κλάσης. Για έναν δυαδικό ταξινομητή με C_FN ≠ C_FP:

```python
# PyTorch
weight = torch.tensor([C_FP, C_FN])
loss_fn = nn.CrossEntropyLoss(weight=weight)

# scikit-learn
from sklearn.linear_model import LogisticRegression
model = LogisticRegression(class_weight={0: C_FP, 1: C_FN})
```

Αυτό προκαταλαμβάνει το μοντέλο να ελαχιστοποιεί το πιο δαπανηρό σφάλμα κατά την εκπαίδευση, μετατοπίζοντας το πραγματικό κατώφλι.

### 4A.3.2 Βαθμονομημένοι Κανόνες Απόφασης

Δεδομένου ενός καλά βαθμονομημένου μοντέλου με έξοδο p(x) = P(y=1|x):

```python
def optimal_decision(score, c_fp, c_fn, base_rate):
    """
    Λήψη βέλτιστης απόφασης δεδομένης της δομής κόστους.
    
    Επιστρέφει: 'positive', 'negative', ή 'uncertain' (για δρομολόγηση HITL)
    """
    # Αναμενόμενο κόστος πρόβλεψης θετικού σε αυτή τη βαθμολογία
    # (P(y=0|x) * C_FP έναντι P(y=1|x) * C_FN)
    p_positive = score  # Βαθμονομημένη βαθμολογία = P(y=1|x)
    p_negative = 1 - score
    
    cost_predict_positive = p_negative * c_fp
    cost_predict_negative = p_positive * c_fn
    
    # Βέλτιστο κατώφλι: βαθμολογία όπου cost_positive = cost_negative
    tau_optimal = c_fp / (c_fp + c_fn)
    
    # Ζώνη HITL: προσθήκη περιθωρίου αβεβαιότητας γύρω από το βέλτιστο κατώφλι
    margin = 0.1  # Ρυθμιζόμενο
    
    if score >= tau_optimal + margin:
        return 'positive'
    elif score <= tau_optimal - margin:
        return 'negative'
    else:
        return 'uncertain'  # Δρομολόγηση σε ανθρώπινη αναθεώρηση
```

## 4A.4 Συστήματα HITL Πολλαπλών Κατωφλίων

### 4A.4.1 Ταξινόμηση Τριών Κατευθύνσεων

Ένα σύστημα ζώνης HITL χωρίζει τον χώρο των βαθμολογιών σε τρεις περιοχές:

```
[0, τ_L)        → Αυτόματο ΑΡΝΗΤΙΚΟ (υψηλή εμπιστοσύνη για αρνητικό)
[τ_L, τ_H]      → ΑΝΘΡΩΠΙΝΗ ΑΝΑΘΕΩΡΗΣΗ (αβέβαιο)
(τ_H, 1]        → Αυτόματο ΘΕΤΙΚΟ (υψηλή εμπιστοσύνη για θετικό)
```

**Βελτιστοποίηση:** Εύρεση των τ_L και τ_H για την ελαχιστοποίηση του συνολικού κόστους του συστήματος:

```
Cost(τ_L, τ_H) = 
    C_FN · P(s(x) < τ_L | y=1) · P(y=1)           [παραλειπόμενα θετικά]
  + C_FP · P(s(x) > τ_H | y=0) · P(y=0)           [ψευδείς συναγερμοί (αυτοματοποιημένοι)]
  + C_H  · P(τ_L ≤ s(x) ≤ τ_H)                    [κόστος ανθρώπινης αναθεώρησης]
  + C_H_FN · P(ανθρώπινη παράλειψη y=1 στη ζώνη)   [ποσοστό ανθρώπινης παράλειψης στη ζώνη]
  + C_H_FP · P(ανθρώπινος ψευδής συναγερμός στη ζώνη) [ποσοστό ανθρώπινου ψευδώς θετικού στη ζώνη]
```

Όπου C_H είναι το κόστος αναθεώρησης ανά περίπτωση και τα C_H_FN, C_H_FP αντιπροσωπεύουν τα ανθρώπινα σφάλματα.

### 4A.4.2 Περιορισμοί Ικανότητας

Όταν η ανθρώπινη αναθεώρηση έχει περιορισμό ικανότητας (μέγιστες αναθεωρήσεις ανά μονάδα χρόνου = R_max):

```
P(τ_L ≤ s(x) ≤ τ_H) · ρυθμός_άφιξης ≤ R_max
```

Αυτό προσθέτει έναν περιορισμό στη βελτιστοποίηση. Καθώς ο ρυθμός_άφιξης αυξάνεται (περίοδοι αιχμής), η ζώνη (τ_H - τ_L) πρέπει να στενέψει — λιγότερες περιπτώσεις πηγαίνουν για ανθρώπινη αναθεώρηση, περισσότερες αντιμετωπίζονται αυτόνομα. Αυτή είναι η μαθηματική βάση για την παρατήρηση στο Κεφάλαιο 4 ότι οι ρυθμίσεις των κατωφλίων πρέπει να προσαρμόζονται στην ικανότητα των αναθεωρητών.
